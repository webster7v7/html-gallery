WSL 简介与核心概念

什么是 WSL？

WSL (Windows Subsystem for Linux) 是 Windows 自带的 Linux 子系统， 允许开发者在 Windows 上原生运行 Linux 环境，无需传统虚拟机的性能开销或双系统启动的繁琐切换。

🐧

原生 Linux

真正的 Linux 内核，完整系统调用兼容性

⚡

极速启动

毫秒级启动，比传统虚拟机快数十倍

🔗

无缝集成

Windows 与 Linux 文件系统、网络互通

核心功能

  • ✓ 在 Windows 终端中直接运行 Linux 命令和 Bash 脚本
  • ✓ 在 Linux 环境中调用 Windows 应用程序（双向互操作）
  • ✓ 原生支持 Docker 容器运行，无需 Docker Desktop 亦可
  • ✓ GPU 直通支持，可直接使用 NVIDIA CUDA 进行 AI 训练
  • ✓ WSLg 支持 Linux GUI 应用，以原生 Windows 窗口形式运行
2

WSL1 vs WSL2 架构差异

WSL1

翻译层架构

本质：一个系统调用翻译层，将 Linux 系统调用实时转换为 Windows NT 内核 API。

无真实 Linux 内核，通过 Windows 内核模拟
文件系统性能差（跨系统访问慢）
无法运行 Docker（缺乏 cgroups 支持）
维护翻译层工作量巨大，兼容性受限

WSL2

虚拟化架构

本质：基于 Hyper-V 的轻量级虚拟机，运行真正的 Linux 内核。

真正的 Linux 内核（微软定制版）
与 Windows 用户空间进行网络通信、文件共享
原生支持 Docker 容器
Hyper-V 硬件虚拟化，自带 GPU 直通
启动时间 < 1 秒，资源占用极低

架构原理：开启 Hyper-V 后，Windows 本身成为一个运行在 Hyper-V 上的"大号虚拟机"， WSL2 是 Hyper-V 上运行的另一个独立的轻量级虚拟机。两者通过虚拟化技术实现高效隔离与通信。

WSL2 安装与配置

1 开启 CPU 虚拟化

大部分电脑默认已开启，请按以下步骤检查：

检查方法：任务管理器 → 性能 → CPU → 查看"虚拟化"状态是否为"已启用"

若未开启：重启进入 BIOS，开启 Intel VT-x (Intel) 或 AMD-V (AMD)

2 开启 Windows 功能

搜索"启用或关闭 Windows 功能"，勾选以下两项：

  • ✓ 适用于 Linux 的 Windows 子系统
  • ✓ 虚拟机平台

点击确定后必须重启电脑才能生效

3 安装默认发行版 (Ubuntu)

管理员命令提示符 / PowerShell 
# 默认安装（国内网络可能较慢）
wsl --install

# 推荐：使用 web-download 模式减少网络问题
wsl --install --web-download

# 安装完成后重启电脑
restart-computer

首次启动配置：安装完成后会自动启动 Ubuntu，提示设置 用户名 和 密码（输入密码时不显示字符）

4

管理 WSL 发行版

命令 功能说明 示例
wsl --list --online 查看可安装的发行版列表 -
wsl --install -d <名称> 安装指定发行版 wsl --install -d Kali-Linux
wsl --list --verbose 查看已安装的发行版及状态
* 号表示默认发行版		 wsl -l -v
wsl --set-default <名称> 设置默认启动的发行版 wsl --set-default Ubuntu
wsl -d <名称> 启动指定发行版 wsl -d Ubuntu-22.04
wsl --terminate <名称> 停止指定发行版 wsl -t Ubuntu
wsl --unregister <名称> 卸载发行版（数据不可恢复） wsl --unregister Ubuntu

提示：Windows Terminal 新版支持直接点击顶部下拉列表选择发行版启动，无需记忆命令。

5

备份、恢复与迁移

💾 导出（备份）

导出为 tar 文件 
wsl --export Ubuntu D:\Backup\ubuntu.tar

将系统导出为 tar 压缩包，包含所有文件和配置

📦 导入（恢复）

从 tar 导入到新位置 
wsl --import Ubuntu-New D:\WSL\Ubuntu-New D:\Backup\ubuntu.tar --version 2

可指定新名称和安装路径，实现系统迁移

完整迁移流程（C盘 → D盘）

1
停止 WSL 
wsl --shutdown
2
导出系统 
mkdir D:\WSL
wsl --export Ubuntu D:\WSL\ubuntu.tar
3
注销原系统 
wsl --unregister Ubuntu
4
导入到新位置 
wsl --import Ubuntu D:\WSL\Ubuntu D:\WSL\ubuntu.tar --version 2
5
设置默认用户 
Ubuntu config --default-user <你的用户名>

文件系统互通

🐧→🪟 Linux 访问 Windows

Windows 磁盘自动挂载到 /mnt/ 目录：

cd /mnt/c/Users/用户名/Desktop  # C盘桌面
cd /mnt/d/Projects             # D盘项目目录
ls /mnt/c

注意：通过挂载目录进行大量 I/O 操作性能较差，建议将项目文件复制到 Linux 内部（/home/用户名/）进行编译等操作

🪟→🐧 Windows 访问 Linux

打开"此电脑"，左侧导航栏会显示 Linux 图标：

📁 展开"Linux"图标查看所有发行版
🖱️ 直接在资源管理器中浏览、编辑文件
或在 WSL 中快速打开资源管理器 
explorer.exe .

高级特性与"黑科技"

命令互操作

在 Linux 中运行 Windows 程序：

notepad.exe
explorer.exe .
code .  # VS Code

在 PowerShell 中使用 Linux 命令处理：

Get-ChildItem | wsl grep video
ipconfig | wsl grep IPv4

WSLg：Linux GUI 应用支持

基于 RDP 协议，在 Windows 窗口中运行 Linux 图形应用：

安装并运行 GIMP 
sudo apt update
sudo apt install gimp -y
gimp  # 直接在 Windows 窗口中打开

无需额外配置，WSL2 默认支持 GUI 应用转发

GPU 直通（CUDA 支持）

WSL2 可直接识别并使用宿主机的 NVIDIA GPU，运行 AI 大模型、深度学习训练：

nvidia-smi  # 查看显卡信息
# 安装 CUDA Toolkit 后即可运行 PyTorch/TensorFlow

需安装 NVIDIA CUDA on WSL 驱动，官网下载：https://developer.nvidia.com/cuda/wsl

Kali Linux 专属：KEX 桌面

安装 Kali Win-KeX 
sudo apt install kali-win-kex
kex --esm --ip --sound  # 启动完整桌面环境

通过远程桌面获得 Kali 完整桌面环境，方便使用网络渗透工具

高级配置

两种配置文件

.wslconfig（全局）

位置：C:\Users\<用户名>\.wslconfig

对所有 WSL2 发行版生效的全局设置

wsl.conf（专有）

位置：/etc/wsl.conf（Linux 内部）

仅对当前 Linux 发行版生效

8 秒规则：修改配置文件后，必须执行 wsl --shutdown 关闭所有实例， 等待 8 秒以上再启动，配置才能生效。

启用 systemd（推荐）

Ubuntu 22.04+ 支持 systemd，可管理服务（如 Docker、MySQL）：

编辑 /etc/wsl.conf 
sudo nano /etc/wsl.conf

# 添加以下内容：
[boot]
systemd=true
# 重启生效
wsl --shutdown
# 等待 8 秒后重新启动 WSL

# 验证
systemctl list-unit-files --type=service

镜像网络模式（Win11 22H2+）

默认 NAT 模式下 WSL 与 Windows 不在同一网段，改为镜像模式可共享 IP：

编辑 ~/.wslconfig 
[wsl2]
networkingMode=mirrored  # 镜像网络模式

效果：WSL 与 Windows 共享同一 IP 地址，局域网其他设备可直接访问 WSL 服务

资源限制配置

~/.wslconfig 完整示例 
[wsl2]
memory=4GB              # 最大内存限制
processors=2            # CPU 核心数限制
swap=2GB                # 交换空间大小
localhostForwarding=true

# Win11 高级特性
[experimental]
autoMemoryReclaim=gradual  # 自动回收内存
sparseVhd=true             # 稀疏 VHD 节省空间

Docker 集成

方案一：Docker Desktop（推荐新手）

官方客户端，自动集成 WSL2 后端：

  1. 下载 Docker Desktop for Windows
  2. 安装时勾选"Use WSL 2 instead of Hyper-V"
  3. Settings → Resources → WSL Integration → 启用你的发行版

方案二：Docker Engine（轻量级）

直接在 WSL2 中安装 Docker，无 GUI 界面，资源占用更低：

一键安装脚本 
curl -fsSL https://get.docker.com | sudo sh

# 或使用国内镜像加速
curl -fsSL https://get.docker.com | sudo sh -s -- --mirror Aliyun
配置用户权限 
sudo usermod -aG docker $USER
newgrp docker  # 立即生效
docker run hello-world  # 验证

配置国内镜像加速

创建/编辑 daemon.json 
sudo mkdir -p /etc/docker
sudo nano /etc/docker/daemon.json
添加镜像源（阿里云/网易云/中科大） 
{
  "registry-mirrors": [
    "https://<你的ID>.mirror.aliyuncs.com",
    "https://hub-mirror.c.163.com",
    "https://docker.mirrors.ustc.edu.cn"
  ]
}

阿里云专属地址获取：https://cr.console.aliyun.com/
