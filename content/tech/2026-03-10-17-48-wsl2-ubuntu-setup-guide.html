前置检查

系统要求

  • ✓ Windows 10: 版本 2004 及更高版本（内部版本 19041 及更高）
  • ✓ Windows 11: 所有版本均支持
  • ✓ 硬件虚拟化: 需在 BIOS 中启用 Intel VT-x 或 AMD-V

检查虚拟化是否启用：打开任务管理器 → 性能 → CPU，查看"虚拟化"是否显示为"已启用"

PowerShell (管理员) 
systeminfo | findstr /i "Hyper-V 虚拟机"
# 或查看详细系统信息
systeminfo
2

安装 WSL2

注意：以下命令需要在 管理员权限的 PowerShell 中执行

方法一：一键自动安装（推荐）

PowerShell (管理员) 
wsl --install
# 安装完成后重启电脑
restart-computer

此命令会自动启用所需功能、安装 WSL2 内核、设置默认版本为 WSL2，并安装默认的 Ubuntu 发行版

方法二：手动分步安装

步骤 1: 启用 WSL 功能 
dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart
步骤 2: 启用虚拟机平台 
dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart
步骤 3: 设置 WSL2 为默认版本 
wsl --set-default-version 2

验证安装：执行 wsl --version 或 wsl -v 查看版本信息

3

安装 Ubuntu 24.04

查看可用发行版 
wsl --list --online
安装 Ubuntu 24.04 LTS 
wsl --install -d Ubuntu-24.04

首次启动配置

安装完成后，首次启动 Ubuntu 24.04 会提示创建用户：

  1. 输入 UNIX 用户名（建议小写字母，如：webster）
  2. 设置 用户密码（输入时不会显示字符）
  3. 确认密码

提示：设置完成后，系统会自动登录。后续可通过开始菜单搜索 "Ubuntu" 或在 PowerShell 中执行 wsl 进入

更换国内镜像源

重要：Ubuntu 24.04 使用新的 DEB822 格式源配置文件，路径为 /etc/apt/sources.list.d/ubuntu.sources，不再是传统的 sources.list

阿里云镜像源（推荐）

备份并编辑源文件 
# 备份原文件
sudo cp /etc/apt/sources.list.d/ubuntu.sources /etc/apt/sources.list.d/ubuntu.sources.bak

# 使用 nano 编辑（或 vim）
sudo nano /etc/apt/sources.list.d/ubuntu.sources

替换为以下内容

DEB822 格式 - 阿里云 
Types: deb
URIs: https://mirrors.aliyun.com/ubuntu/
Suites: noble noble-updates noble-backports
Components: main restricted universe multiverse
Signed-By: /usr/share/keyrings/ubuntu-archive-keyring.gpg

Types: deb
URIs: https://mirrors.aliyun.com/ubuntu/
Suites: noble-security
Components: main restricted universe multiverse
Signed-By: /usr/share/keyrings/ubuntu-archive-keyring.gpg
阿里云
mirrors.aliyun.com
清华大学
mirrors.tuna.tsinghua.edu.cn
中科大
mirrors.ustc.edu.cn
更新源并升级 
sudo apt update
sudo apt upgrade -y

迁移至非C盘

注意：默认 WSL 安装在 C 盘，随着使用会占用大量空间。建议迁移到 D 盘或其他分区

迁移步骤

步骤 1: 停止 WSL 
wsl --shutdown
步骤 2: 导出系统到 D 盘 
# 创建目录
mkdir D:\WSL\Ubuntu24

# 导出为 tar 格式（或 vhdx）
wsl --export Ubuntu-24.04 D:\WSL\Ubuntu24\ubuntu2404.tar
步骤 3: 注销原系统 
wsl --unregister Ubuntu-24.04
步骤 4: 导入到新位置 
wsl --import Ubuntu-24.04 D:\WSL\Ubuntu24 D:\WSL\Ubuntu24\ubuntu2404.tar --version 2
步骤 5: 设置默认用户 
# 将 username 替换为你的用户名
Ubuntu-24.04 config --default-user username

验证迁移：执行 wsl -l -v 查看系统状态，确认可以正常启动

Docker 集成

安装 Docker Engine

一键安装脚本 
# 使用官方脚本安装
curl -fsSL https://get.docker.com | sudo sh

# 或者使用国内镜像加速安装
curl -fsSL https://get.docker.com | sudo sh -s -- --mirror Aliyun

配置用户权限

免 sudo 使用 Docker 
# 将当前用户加入 docker 组
sudo usermod -aG docker $USER

# 重新登录或执行以下命令生效
newgrp docker

# 验证
docker --version
docker run hello-world

配置国内镜像加速

创建/编辑 daemon.json 
sudo mkdir -p /etc/docker
sudo nano /etc/docker/daemon.json
添加阿里云镜像加速 
{
  "registry-mirrors": [
    "https://<你的ID>.mirror.aliyuncs.com",
    "https://docker.mirrors.ustc.edu.cn",
    "https://hub-mirror.c.163.com"
  ]
}

在阿里云控制台获取专属加速地址：https://cr.console.aliyun.com/

性能优化

限制 WSL2 资源使用

在用户目录创建 .wslconfig 文件（如 C:\Users\用户名\.wslconfig）

.wslconfig 
[wsl2]
memory=4GB              # 限制最大内存使用
processors=2            # 限制CPU核心数
swap=2GB                # 设置交换空间
localhostForwarding=true

# 高级网络配置（Win11 22H2+）
[experimental]
autoMemoryReclaim=gradual
sparseVhd=true

清理磁盘空间

Ubuntu 内清理 
# 清理 apt 缓存
sudo apt autoremove -y
sudo apt clean

# 查看磁盘使用
df -h
PowerShell 中压缩虚拟磁盘 
# 关闭 WSL
wsl --shutdown
