📚 网络核心概念

理解这几个核心概念，你就掌握了网络的 80%：

📍 IP 地址

设备在网络中的"门牌号"，每台联网设备都有唯一的 IP 地址，用于互相识别和通信

🎭 子网掩码

划分网络和主机的"分界线"，用来判断两个 IP 是否在同一个网络内

🚪 网关

网络的"大门"，内网设备通过网关才能访问外网（互联网）

📖 DNS

域名系统，把网址（如 www.baidu.com）翻译成 IP 地址的"电话簿"

🔢 MAC 地址

网卡的"身份证号"，由厂商烧录，全球唯一，用于局域网内设备识别

🔌 端口

设备上的"服务窗口"，不同应用使用不同端口（如 HTTP 用 80，HTTPS 用 443）

🏠 生活化比喻

想象一栋公寓大楼：IP 地址是房间号（如 3 栋 502），MAC 地址是你的身份证号，网关是公寓大门，DNS是快递查询系统（知道"小明家"对应 502 房间），端口是房间里的不同功能区域（厨房、卧室、客厅）。

📍 IP 地址详解

IPv4 vs IPv6

特性 IPv4 IPv6
地址长度 32 位 128 位
地址格式 192.168.1.1 2001:0db8:85a3::8a2e:0370:7334
地址数量 约 43 亿个 几乎无限（2^128）
普及程度 目前主流 正在普及

IP 地址分类

类型 范围 用途
公网 IP 除私有地址外 互联网上的唯一标识
私有 IP（A类） 10.0.0.0 ~ 10.255.255.255 大型企业内网
私有 IP（B类） 172.16.0.0 ~ 172.31.255.255 中型企业内网
私有 IP（C类） 192.168.0.0 ~ 192.168.255.255 家庭/小型办公（最常见）
回环地址 127.0.0.1 本机测试（localhost）

💡 小知识

  • 你在家看到的 192.168.x.x 是私有 IP，只在你的家庭网络内有效
  • 访问互联网时，路由器会把私有 IP 转换成公网 IP（NAT 技术）
  • 查询自己的公网 IP：百度搜索"IP"或在命令行输入 curl ip.sb

🔌 常见网络设备

📶

路由器（Router）

连接不同网络的设备，负责数据包的转发和路由选择，是家庭网络的核心

🔀

交换机（Switch）

连接局域网内的设备，实现设备间的数据交换，比集线器更智能高效

📡

光猫（Modem）

光电信号转换设备，把光纤的光信号转换成电信号，是接入互联网的入口

📳

AP（无线接入点）

提供 WiFi 覆盖的设备，可以单独购买，现在很多路由器自带 AP 功能

🔧

集线器（Hub）

早期的网络设备，简单广播数据，已被交换机取代，现在很少见

🛡️

防火墙

网络安全设备，控制进出网络的流量，阻挡非法访问和攻击

🏢 公司比喻

路由器是前台，知道包裹该送到哪层楼；交换机是各层的收发室，负责把包裹送到具体办公室；光猫是快递公司的货车，负责和外部世界连接。

两种上网模式

🔧 光猫拨号（路由模式）

光猫负责拨号上网，路由器接在光猫下面作为二级路由。适合不懂网络设置的用户，但光猫性能可能较弱

⚡ 路由器拨号（桥接模式）

光猫设置为桥接模式，路由器负责拨号。性能更好，但需要配置，推荐有一定网络知识的用户使用

💡 桥接模式设置步骤

  1. 获取宽带账号密码（打电话问运营商）
  2. 登录光猫管理页面（通常是 192.168.1.1）
  3. 找到"网络"或"宽带设置"，将路由模式改为桥接模式
  4. 登录路由器，设置上网方式为"PPPoE 拨号"
  5. 输入宽带账号密码，保存重启

⌨️ 网络排查命令

Windows 命令

CMD / PowerShell
# 查看 IP 配置
C:\> ipconfig
# 查看详细 IP 配置（包括 MAC 地址）
C:\> ipconfig /all
# 测试网络连通性
C:\> ping www.baidu.com
# 追踪数据包路由路径
C:\> tracert www.baidu.com
# 查看网络连接状态
C:\> netstat -an
# 刷新 DNS 缓存
C:\> ipconfig /flushdns
# 释放 IP 地址
C:\> ipconfig /release
# 重新获取 IP 地址
C:\> ipconfig /renew

Mac/Linux 命令

Terminal
# 查看 IP 配置（Mac）
$ ifconfig
# 查看 IP 配置（Linux）
$ ip addr
# 测试网络连通性
$ ping www.baidu.com
# 追踪路由（Mac）
$ traceroute www.baidu.com
# 追踪路由（Linux）
$ tracepath www.baidu.com
# 查询 DNS
$ nslookup www.baidu.com
# 查看网络连接
$ netstat -tuln
# 测试端口连通性
$ nc -zv 192.168.1.1 80

命令详解

命令 作用 使用场景
ping 测试与目标主机的连通性 判断能否上网、网络是否稳定
ipconfig/ifconfig 查看本机网络配置 查看 IP、网关、DNS 等信息
tracert/traceroute 追踪数据包路由路径 定位网络故障点
nslookup 查询域名对应的 IP 排查 DNS 问题
netstat 查看网络连接状态 查看端口占用情况

🏠 家庭组网指南

路由器选购建议

户型 推荐方案 预算参考
小户型（<80㎡） 单台路由器（WiFi 6） 200-400元
中户型（80-120㎡） Mesh 组网（2台） 500-1000元
大户型（>120㎡） AC+AP 或 Mesh（3台+） 1000-3000元
复式/别墅 AC+AP 专业方案 3000元+

💡 路由器选购要点

  • WiFi 6（802.11ax）：新一代标准，速度更快，延迟更低，建议优先选择
  • 千兆网口：WAN 口和 LAN 口都要支持千兆，否则宽带跑不满
  • 5G 频段：双频路由器必备，5G 速度快但穿墙差，2.4G 穿墙好但速度慢
  • 天线数量：不是越多越好，看实际需求和空间覆盖

WiFi 设置建议

  • 修改路由器管理密码

    不要使用默认的 admin/admin，设置强密码防止被蹭网

  • 设置 WiFi 名称（SSID）

    建议使用英文或数字，避免中文兼容性问题。可以设置 2.4G 和 5G 不同名称，方便选择

  • 设置 WiFi 密码

    使用 WPA2/WPA3 加密，密码至少 8 位，包含大小写字母和数字

  • 选择信道

    2.4G 建议选择 1、6、11 信道（互不干扰），5G 可以选自动

  • 开启访客网络

    给客人使用，隔离主网络，保护隐私和设备安全

优化 WiFi 信号

📍 放置位置

放在房屋中心位置，远离金属物品、微波炉、蓝牙设备等干扰源

📐 天线角度

多根天线不要全部垂直，可以 45° 错开，覆盖更均匀

🔧 定期重启

路由器长期运行会卡顿，建议每周重启一次，或设置自动重启

📶 信号强度

WiFi 信号不是越强越好，适当降低功率可以减少干扰

🔧 常见故障排查

无法上网排查流程

  • 第一步：检查物理连接

    查看网线是否插好，光猫和路由器的指示灯是否正常（绿灯正常，红灯异常）

  • 第二步：检查 IP 地址

    运行 ipconfig（Windows）或 ifconfig（Mac），查看是否获取到 192.168.x.x 的 IP

  • 第三步：Ping 网关

    ping 路由器的 IP（通常是 192.168.1.1），不通说明电脑到路由器有问题

  • 第四步：Ping 公网 IP

    ping 114.114.114.114（国内 DNS），通说明网络正常，不通说明外网有问题

  • 第五步：Ping 域名

    ping www.baidu.com，如果 IP 通但域名不通，说明 DNS 有问题

常见问题及解决方案

WiFi 能连但无法上网 ▼

可能原因：

  1. 宽带欠费或运营商故障
  2. 光猫或路由器未拨号成功
  3. DNS 设置错误

解决方法：

  1. 拨打运营商客服查询宽带状态
  2. 重启光猫和路由器
  3. 检查路由器 WAN 口状态，确认已获取到公网 IP
  4. 手动设置 DNS 为 223.5.5.5（阿里）或 114.114.114.114
WiFi 信号弱/覆盖差 ▼

解决方法：

  1. 调整路由器位置到房屋中心
  2. 更换信道，避免邻居干扰（使用 WiFi 分析仪 APP 查看）
  3. 增加 Mesh 节点或无线 AP 扩展覆盖
  4. 考虑使用电力猫（通过电线传输网络）
  5. 如果可能，预埋网线做有线 AP
网速慢/卡顿 ▼

排查步骤：

  1. 测速：使用 speedtest.cn 或运营商测速工具，确认是否达到签约带宽
  2. 检查连接设备数：过多设备同时下载会抢占带宽
  3. 检查是否有设备在下载/上传：暂停大流量应用
  4. 切换 WiFi 频段：5G 频段干扰少、速度快，但穿墙差
  5. 更换网线：老旧网线可能只支持百兆
  6. 路由器性能瓶颈：老旧路由器处理能力有限，考虑更换
网页打不开，但 QQ/微信能用 ▼

原因：这是典型的 DNS 故障。QQ/微信使用 IP 直连，不需要 DNS；浏览器访问网页需要 DNS 解析域名。

解决方法：

  1. 刷新 DNS 缓存：Windows 运行 ipconfig /flushdns
  2. 更换 DNS 服务器：
    • 阿里 DNS：223.5.5.5 / 223.6.6.6
    • 腾讯 DNS：119.29.29.29
    • 114 DNS：114.114.114.114
    • Google DNS：8.8.8.8（国内可能不稳定）
  3. 在路由器或电脑上手动设置 DNS
能上网但经常掉线 ▼

可能原因及解决：

  1. 路由器过热：检查散热，必要时加风扇或更换
  2. IP 地址冲突：重启路由器重新分配 IP，或设置静态 IP
  3. 光猫不稳定：联系运营商检查或更换光猫
  4. 路由器性能不足：连接设备过多，考虑升级
  5. 信道干扰：更换 WiFi 信道

🛡️ 网络安全建议

基础安全措施

🔑 修改默认密码

路由器管理密码、WiFi 密码都要修改，不要使用默认的 admin

🔒 启用 WPA3

选择 WPA3 或 WPA2 加密，不要使用已不安全的 WEP

🚫 关闭 WPS

WPS 功能有安全漏洞，建议关闭，手动输入密码更安全

👥 开启访客网络

访客网络与主网络隔离，保护你的设备和数据安全

进阶安全设置

  • 关闭远程管理

    除非必要，关闭路由器的远程管理功能，防止被攻击

  • 定期更新固件

    及时更新路由器固件，修复安全漏洞

  • 关闭 SSID 广播（可选）

    隐藏 WiFi 名称，知道名称才能连接，增加安全性

  • 启用 MAC 地址过滤

    只允许指定设备连接，但 MAC 地址可以伪造，不是绝对安全

  • 定期检查连接设备

    在路由器管理页面查看连接设备列表，发现可疑设备及时踢出

⚠️ 安全提醒

  • 不要使用公共场所的免费 WiFi 进行网银操作
  • 家中的 WiFi 密码不要告诉陌生人
  • 定期更换 WiFi 密码，特别是发现网速异常时
  • 智能设备（摄像头、智能音箱等）尽量使用访客网络
