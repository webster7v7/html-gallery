1

配置 Android Termux 服务端

在手机上搭建 SSH 服务器，开放远程访问权限

# 更新软件源并安装 OpenSSH
pkg update && pkg upgrade -y
pkg install openssh -y

# 设置登录密码（必须执行）
passwd

# 启动 SSH 服务（默认端口 8022）
sshd

# 查看当前用户名（连接时需要）
whoami
# 输出示例: u0_a123
获取 IP 地址
ifconfig 或 ip addr
查找 wlan0 下的 inet 地址，如 192.168.1.5
后台保活设置
执行 termux-wake-lock 防止进程被杀
建议将 Termux 加入电池优化白名单

WiFi 局域网连接（推荐）

1. 确保 Windows 11 和 Android 连接同一 WiFi 网络

2. 在 Termux 中获取手机 IP: ifconfig

3. 在 Termius 中新建主机，填入 IP 和端口 8022

连接参数示例:
Host: 192.168.1.5:8022
Username: u0_a123
Password: [你设置的密码]

USB ADB 转发（无需 WiFi）

1. 开启手机 USB 调试模式（开发者选项）

2. Windows 安装 ADB 工具（通过 Android Studio 或单独下载）

# 在 Windows PowerShell/CMD 执行
adb forward tcp:8022 tcp:8022

# 然后 Termius 连接 localhost:8022 即可

* 优势：不依赖无线网络，数据传输更稳定安全

手机热点直连

1. Android 开启个人热点

2. Windows 11 连接该热点

3. 在 Termux 中查看热点分配的 IP（通常是 192.168.43.x 或 172.20.10.x）

# 查看路由信息获取网关 IP
ip route | grep default
3

Windows Termius 客户端配置

现代 SSH 客户端的专业级连接体验 [^4^]

1 新建主机配置

Label My Android Phone
Address 192.168.x.x:8022
Username u0_a123
Password ••••••••

💡 高级功能：建议启用"Sync across devices"以在多设备间同步此配置

2 密钥认证（更安全）

替代密码登录，使用 ED25519 密钥对 [^9^]

# 在 Termux 生成密钥
ssh-keygen -t ed25519 -C "termux"

# 查看并复制公钥内容
cat ~/.ssh/id_ed25519.pub

# 授权公钥
cat id_ed25519.pub >> ~/.ssh/authorized_keys

将私钥导入 Termius Keychain，禁用密码登录更安全

🚀 进阶优化技巧

自动启动 SSH 服务

# 添加到 ~/.bashrc，每次启动自动运行
echo "sshd" >> ~/.bashrc

# 或者创建启动脚本
echo "sshd -p 8022" > ~/.termux/boot/sshd
chmod +x ~/.termux/boot/sshd

固定 IP 配置（可选）

避免每次查看 IP，可在路由器设置静态 DHCP 绑定，或在 Termux 使用静态 IP：

# 需要 root 或使用 adb shell
ifconfig wlan0 192.168.1.100 netmask 255.255.255.0

⚠️ 安全建议

  • 避免在公共 WiFi 下开放 SSH 服务
  • 定期更换密码或使用密钥认证
  • 可通过 who 命令查看当前登录用户
  • 使用 pkill sshd 随时停止服务

🛠️ 常见问题排查

• 检查手机和电脑是否在同一网络（ping 测试）

• 确认 Termux 中 sshd 已运行（ps aux | grep sshd）

• 检查 Android 防火墙/省电模式是否限制 Termux 联网

• 尝试切换连接方式（WiFi 转 USB 或热点）

• 重新在 Termux 执行 passwd 设置密码

• 注意 Termux 用户名格式为 u0_a数字，不是 root

• 检查 Termius 中用户名是否包含多余空格

• 尝试使用密钥认证替代密码

• 执行 termux-wake-lock 保持后台运行

• 手机设置 → 电池 → 忽略 Termux 的电池优化

• 锁定 Termux 应用（多任务界面下拉锁定）

• 关闭手机"智能省电"功能
