👋 Nginx是什么？

🎯 一句话理解

Nginx 是一个高性能的 Web服务器 和 反向代理服务器。简单来说，它是网站的"门卫"和"交通指挥员"。

生活化比喻

想象一个大型商场：

  • 🏢 Nginx = 商场的总服务台 + 安保系统
  • 👥 用户 = 来商场的顾客
  • 🏪 后端服务器 = 商场里的各个店铺

Nginx 的工作：

  • 📍 指路：告诉顾客（用户）去哪里找店铺（网站）
  • 🛡️ 安保：拦截可疑人员（恶意请求）
  • ⚡ 分流：高峰期引导顾客去人少的店铺（负载均衡）
  • 💾 缓存：记住常问的问题，直接回答（缓存静态资源）

Nginx能做什么？

功能 说明 使用场景
🌐 静态资源服务 直接提供HTML、CSS、JS、图片等文件 部署个人博客、企业官网
🔄 反向代理 接收用户请求，转发给后端服务器 保护后端服务，统一入口
⚖️ 负载均衡 将请求分配到多台服务器 高并发网站，如电商、社交平台
🔒 HTTPS SSL证书配置，加密传输 任何需要安全传输的网站
📦 缓存 缓存静态内容，加速访问 减轻后端压力，提升速度
💡 为什么选择Nginx？

1. 速度快：能同时处理数万个连接（并发能力强）
2. 省资源：占用内存少，一台普通电脑就能跑
3. 稳定：很多大公司（百度、淘宝、腾讯）都在用
4. 免费：开源软件，完全免费

🛠️ 安装Nginx

根据你的操作系统选择对应的安装方法：

Windows安装步骤

下载Nginx

访问官网下载页面：nginx.org

下载 Stable version（稳定版）的 Windows 版本，如 nginx-1.24.0.zip

解压文件

将下载的zip文件解压到任意目录，建议放在 C:\nginx

⚠️ 注意：路径不要有中文或空格！

启动Nginx

打开命令提示符（CMD），进入nginx目录：

CMD 
cd C:\nginx
start nginx
验证安装

打开浏览器，访问 http://localhost

如果看到 "Welcome to nginx!" 页面，说明安装成功！🎉

⚠️ Windows常见问题

如果启动失败，可能是端口80被占用。检查方法：

CMD 
netstat -ano | findstr :80

如果被占用，需要修改配置文件（后面会讲）。

⚙️ 配置文件详解

配置文件位置

系统 主配置文件路径 默认网站目录
Windows C:\nginx\conf\nginx.conf C:\nginx\html
Linux /etc/nginx/nginx.conf /var/www/html
Mac /opt/homebrew/etc/nginx/nginx.conf /opt/homebrew/var/www

最小可用配置

下面是一个最简化的Nginx配置，小白可以直接复制使用：

nginx.conf 
# 工作进程数，通常等于CPU核心数
worker_processes 1;

# 错误日志位置
error_log logs/error.log;

# 主进程PID文件
pid logs/nginx.pid;

events {
    # 每个工作进程的最大连接数
    worker_connections 1024;
}

http {
    # 包含mime类型文件
    include mime.types;
    
    # 默认类型
    default_type application/octet-stream;
    
    # 日志格式
    log_format main '$remote_addr - $remote_user [$time_local] "$request" '
                    '$status $body_bytes_sent "$http_referer" '
                    '"$http_user_agent" "$http_x_forwarded_for"';
    
    # 访问日志
    access_log logs/access.log main;
    
    # 开启高效传输模式
    sendfile on;
    
    # 长连接超时时间
    keepalive_timeout 65;
    
    # 服务器配置
    server {
        # 监听80端口
        listen 80;
        
        # 服务器名称（域名）
        server_name localhost;
        
        # 网站根目录
        root html;
        
        # 默认首页文件
        index index.html index.htm;
        
        #  location块：处理所有请求
        location / {
            # 尝试按顺序查找文件，找不到返回index.html
            try_files $uri $uri/ /index.html;
        }
        
        # 错误页面
        error_page 500 502 503 504 /50x.html;
        location = /50x.html {
            root html;
        }
    }
}

配置指令解释

指令 说明 示例
listen 监听的端口 listen 80; listen 8080;
server_name 服务器域名 server_name example.com;
root 网站文件根目录 root /var/www/html;
index 默认首页 index index.html index.php;
location 匹配URL路径 location /images/ { ... }

🌐 部署静态网站

静态网站是指由HTML、CSS、JavaScript、图片等固定文件组成的网站，不需要后端程序处理。

实战：部署个人主页

准备网站文件

创建一个简单的HTML文件作为示例：

index.html 
<!DOCTYPE html>
<html>
<head>
    <title>我的个人主页</title>
    <style>
        body {
            font-family: Arial, sans-serif;
            max-width: 800px;
            margin: 50px auto;
            padding: 20px;
            background: linear-gradient(135deg, #667eea 0%, #764ba2 100%);
            color: white;
        }
        h1 { text-align: center; }
    </style>
</head>
<body>
    <h1>欢迎来到我的网站！</h1>
    <p>这是由Nginx提供服务的静态网站。</p>
    <p>部署时间：<span id="time"></span></p>
    <script>
        document.getElementById('time').innerText = new Date().toLocaleString();
    </script>
</body>
</html>
放置文件

将 index.html 放到Nginx的html目录：

  • Windows: C:\nginx\html\index.html
  • Linux: /var/www/html/index.html
  • Mac: /opt/homebrew/var/www/index.html
访问网站

浏览器访问 http://localhost 或服务器的IP地址

你应该能看到刚才创建的彩色页面！

修改网站目录

如果你想把网站放在其他位置（比如D盘的website文件夹），需要修改配置：

nginx.conf 
server {
    listen 80;
    server_name localhost;
    
    # Windows示例
    root D:/website;
    
    # Linux/Mac示例
    # root /home/user/website;
    
    index index.html;
    
    location / {
        try_files $uri $uri/ =404;
    }
}
⚠️ Windows路径注意

Windows路径既可以用斜杠 D:/website 也可以用反斜杠 D:\\website

但用斜杠更符合Nginx的习惯，也更不容易出错。

🔄 反向代理配置

💡 什么是反向代理？

反向代理就是：用户只访问Nginx，Nginx再把请求转发给真正的后端服务器（如Node.js、Python、Java程序）。

好处：隐藏后端服务器、统一入口、负载均衡、SSL加密。

场景1：代理到本地后端服务

假设你有一个Node.js服务运行在 http://localhost:3000

nginx.conf 
server {
    listen 80;
    server_name localhost;
    
    # 所有请求都代理到后端
    location / {
        proxy_pass http://localhost:3000;
        
        # 设置请求头（让后端知道真实IP和协议）
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # 超时设置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }
}

场景2：API和静态资源分离

静态文件由Nginx直接提供，API请求转发给后端：

nginx.conf 
server {
    listen 80;
    server_name localhost;
    
    # 静态文件（前端）
    location / {
        root /var/www/frontend;
        index index.html;
        try_files $uri $uri/ /index.html;
    }
    
    # API请求（转发给后端）
    location /api/ {
        proxy_pass http://localhost:8080/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
    
    # 图片等静态资源（Nginx直接处理，更快）
    location /images/ {
        root /var/www/static;
        expires 30d;  # 缓存30天
    }
}
💡 location匹配规则

= 精确匹配，如 location = /exact

^~ 前缀匹配，不匹配正则

~ 正则匹配（区分大小写）

~* 正则匹配（不区分大小写）

🌐 配置多域名（虚拟主机）

一台服务器可以托管多个网站，每个网站使用不同的域名。

配置多个server块

nginx.conf 
http {
    # 第一个网站：个人博客
    server {
        listen 80;
        server_name blog.example.com;
        
        root /var/www/blog;
        index index.html;
        
        location / {
            try_files $uri $uri/ =404;
        }
    }
    
    # 第二个网站：公司官网
    server {
        listen 80;
        server_name www.example.com;
        
        root /var/www/company;
        index index.html index.htm;
        
        location / {
            try_files $uri $uri/ =404;
        }
    }
    
    # 第三个网站：API服务
    server {
        listen 80;
        server_name api.example.com;
        
        location / {
            proxy_pass http://localhost:3000;
        }
    }
}
📁 更好的组织方式

把每个网站的配置放在单独的文件里，更易于管理：

/etc/nginx/ ├── nginx.conf # 主配置 ├── conf.d/ # 存放各个网站的配置 │ ├── blog.conf # 博客配置 │ ├── company.conf # 公司官网配置 │ └── api.conf # API配置 └── sites-enabled/ # 启用的网站（软链接）

在 nginx.conf 的 http 块中加入：

include /etc/nginx/conf.d/*.conf;

🔒 HTTPS配置（SSL证书）

HTTPS能让网站数据传输加密，防止被窃听。现在主流浏览器都会标记HTTP网站为"不安全"。

使用Let's Encrypt免费证书

安装Certbot

Certbot是Let's Encrypt的官方客户端，可以自动申请和续期证书。

Ubuntu/Debian 
# 安装Certbot和Nginx插件
sudo apt install certbot python3-certbot-nginx
申请证书

执行以下命令，按照提示操作：

Terminal 
# 自动配置Nginx并申请证书
sudo certbot --nginx -d example.com -d www.example.com

# 按照提示输入邮箱、同意协议
自动续期

Let's Encrypt证书有效期90天，但Certbot会自动续期。

测试自动续期是否正常：

Terminal 
sudo certbot renew --dry-run

手动配置SSL（已有证书）

如果你已经从其他渠道获得了证书文件（.crt和.key），手动配置如下：

nginx.conf 
server {
    listen 443 ssl http2;
    server_name example.com;
    
    # SSL证书配置
    ssl_certificate /path/to/your/certificate.crt;
    ssl_certificate_key /path/to/your/private.key;
    
    # SSL优化配置
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-RSA-AES128-GCM-SHA256:ECDHE-RSA-AES256-GCM-SHA384;
    ssl_prefer_server_ciphers off;
    
    # HSTS（强制HTTPS）
    add_header Strict-Transport-Security "max-age=63072000" always;
    
    root /var/www/html;
    index index.html;
}

# HTTP重定向到HTTPS
server {
    listen 80;
    server_name example.com;
    return 301 https://$server_name$request_uri;
}

🚀 性能优化

开启Gzip压缩

压缩传输内容，减少网络流量，加快页面加载：

nginx.conf 
http {
    # 开启gzip
    gzip on;
    
    # 压缩级别（1-9，数字越大压缩率越高，但越耗CPU）
    gzip_comp_level 6;
    
    # 最小压缩文件大小
    gzip_min_length 1k;
    
    # 压缩类型
    gzip_types text/plain text/css text/javascript application/javascript 
               application/json application/xml image/svg+xml;
    
    # 对代理的内容也压缩
    gzip_proxied any;
    
    # 在响应头添加Vary: Accept-Encoding
    gzip_vary on;
}

浏览器缓存

让浏览器缓存静态资源，减少重复请求：

nginx.conf 
server {
    # 图片缓存30天
    location ~* \\.(jpg|jpeg|png|gif|ico|css|js|svg)$ {
        expires 30d;
        add_header Cache-Control "public, immutable";
    }
    
    # HTML不缓存（总是获取最新）
    location ~* \\.html$ {
        expires -1;
        add_header Cache-Control "no-cache";
    }
}

连接优化

nginx.conf 
events {
    # 使用epoll（Linux高性能网络模型）
    use epoll;
    
    # 每个工作进程的最大连接数
    worker_connections 4096;
    
    # 允许一个工作进程同时接受多个新连接
    multi_accept on;
}

http {
    # 开启文件高效传输
    sendfile on;
    
    # 优化磁盘IO
    tcp_nopush on;
    tcp_nodelay on;
    
    # 长连接保持时间
    keepalive_timeout 30;
    keepalive_requests 1000;
    
    # 重置超时连接
    reset_timedout_connection on;
    
    # 客户端请求体大小限制
    client_body_buffer_size 128k;
    client_max_body_size 50m;
}

🐛 常见问题排查

问题1：端口被占用

错误：bind() to 0.0.0.0:80 failed (10048: Only one usage of each socket address...)

解决：

查找占用端口的进程 
# Windows
netstat -ano | findstr :80
taskkill /PID 1234 /F  # 1234是占用端口的进程ID

# Linux/Mac
sudo lsof -i :80
sudo kill -9 1234

或者修改Nginx监听其他端口，如 listen 8080;

问题2：403 Forbidden（禁止访问）

原因：

  • 网站目录权限不足
  • index文件不存在
  • SELinux限制（Linux）

解决：

Linux权限修复 
# 修改目录权限
sudo chown -R www-data:www-data /var/www/html
sudo chmod -R 755 /var/www/html

# 如果是SELinux问题
sudo setenforce 0  # 临时关闭
sudo sed -i 's/SELINUX=enforcing/SELINUX=disabled/g' /etc/selinux/config

问题3：配置文件错误

测试配置 
# 测试配置文件语法
nginx -t

# 查看详细错误信息
nginx -t -c /path/to/nginx.conf

问题4：无法访问（防火墙）

检查防火墙是否放行了80/443端口：

防火墙设置 
# Ubuntu/Debian (UFW)
sudo ufw allow 'Nginx Full'
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp

# CentOS (Firewalld)
sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --permanent --add-service=https
sudo firewall-cmd --reload

# Windows防火墙
netsh advfirewall firewall add rule name="Nginx HTTP" dir=in action=allow protocol=TCP localport=80
