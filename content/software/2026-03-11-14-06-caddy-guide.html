Caddy 是什么？

Caddy 是一个用 Go 语言 编写的开源 Web 服务器，2015 年由 Matt Holt 开发。它的设计理念是"简单易用"——把复杂的 Web 服务器配置简化到极致。

自动 HTTPS

自动申请、续期 Let's Encrypt 证书，无需手动操作

配置极简

Caddyfile 语法直观，3 行配置即可运行

现代协议

原生支持 HTTP/2、HTTP/3 (QUIC)

Caddyfile 
example.com {
    root * /var/www/html
    file_server
}

✓ 自动 HTTPS 已启用

证书自动申请并续期，无需额外配置

安装 Caddy

支持 Linux、Windows、macOS，推荐使用官方安装脚本

Ubuntu / Debian

Terminal
sudo apt install -y debian-keyring debian-archive-keyring apt-transport-https curl curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/gpg.key' | sudo gpg --dearmor -o /usr/share/keyrings/caddy-stable-archive-keyring.gpg curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/debian.deb.txt' | sudo tee /etc/apt/sources.list.d/caddy-stable.list sudo apt update sudo apt install caddy

CentOS / RHEL

Terminal
sudo dnf install 'dnf-command(copr)' sudo dnf copr enable @caddy/caddy sudo dnf install caddy

Docker (推荐)

Terminal
docker run -d -p 80:80 -p 443:443 --name caddy -v $(pwd)/Caddyfile:/etc/caddy/Caddyfile -v $(pwd):/srv caddy:2

验证安装

caddy version 应显示版本号，如 v2.7.x

5 分钟快速上手

从安装到运行，只需 3 个简单步骤

1

创建 Caddyfile

在项目目录创建名为 Caddyfile 的文件（无后缀），这是 Caddy 的配置文件。

静态网站 
example.com {
    root * /var/www/html
    file_server
}
反向代理 
example.com {
    reverse_proxy localhost:3000
}

提示：将 example.com 替换为你的域名，确保 DNS 解析到服务器 IP

2

放置网站文件

如果是静态网站，将 HTML 文件放入配置的目录：

Bash
sudo mkdir -p /var/www/html echo "<h1>Hello Caddy!</h1>" | sudo tee /var/www/html/index.html
3

启动 Caddy

在 Caddyfile 所在目录运行：

前台运行（调试）
caddy run
后台服务（生产）
sudo systemctl start caddy sudo systemctl enable caddy

🎉 完成！访问你的域名，Caddy 已自动配置 HTTPS，证书自动续期

常用配置场景

复制即用，满足 90% 的需求

静态网站托管

example.com {
    root * /var/www/html
    file_server
    encode gzip
}

自动开启 Gzip 压缩，提升加载速度

反向代理（Node.js/Python）

example.com {
    reverse_proxy localhost:3000
    header_up X-Real-IP {remote}
}

传递真实 IP 给后端应用

多站点托管

example.com {
    root * /var/www/example
    file_server
}

api.example.com {
    reverse_proxy localhost:8080
}

一个 Caddyfile 管理多个域名

路径分流

example.com {
    reverse_proxy /api/* localhost:8080
    reverse_proxy /app/* localhost:3000
    file_server
}

不同路径代理到不同后端服务

Caddy vs Nginx

为什么小白应该选择 Caddy？

特性 Caddy Nginx
HTTPS 配置 ✓ 零配置，自动申请续期 需手动配置 Certbot
配置文件 ✓ 3 行搞定，语法简单 数十行，学习曲线陡峭
HTTP/3 支持 ✓ 原生支持 需额外编译模块
性能 足够快，适合中小项目 ✓ 极高，适合高并发
生态成熟度 较新，插件较少 ✓ 生态完善，文档丰富

建议：个人项目、中小网站、快速原型开发选 Caddy；大型高并发生产环境选 Nginx。

常用命令速查

caddy run 前台运行
caddy start 后台启动
caddy stop 停止服务
caddy reload 重载配置（热更新）
caddy validate 验证配置语法
caddy list-modules 查看已安装模块
