❓ 什么是 Git？

Git 是一个分布式版本控制系统，简单说就是帮你管理代码历史的工具。它能记住你代码的每一次变动，让你可以随时回退到之前的版本，也能让多人协作开发变得轻松。

💾 版本控制

保存代码的历史版本，随时查看、比较、回退到任意时间点

🌐 分布式

每个人的电脑都有完整的代码仓库，不依赖网络也能工作

👥 协作开发

多人同时开发，自动合并代码，解决冲突

🌿 分支管理

并行开发不同功能，互不干扰，灵活切换

💡 为什么学 Git？

  • 代码改崩了可以一键回退
  • 查看自己或别人改了什么
  • 多人协作不会互相覆盖代码
  • 找工作必备技能，99%的公司都在用

⬇️ 安装与配置

选择你的操作系统

  • 下载安装包

    访问官网 git-scm.com/download/win，下载会自动开始

  • 运行安装程序

    双击 .exe 文件，一路点击 "Next" 即可（保持默认设置）

  • 验证安装

    打开命令提示符（CMD）或 PowerShell，输入：

    BASH git --version

    看到版本号（如 git version 2.40.0）说明安装成功！

  • 方法一：通过 Homebrew 安装（推荐）

    如果你已经安装了 Homebrew，直接在终端输入：

    BASH brew install git

  • 方法二：下载安装包

    访问 git-scm.com/download/mac 下载安装包

  • 验证安装

    BASH git --version

  • Ubuntu/Debian

    BASH sudo apt-get update
    sudo apt-get install git

  • CentOS/RHEL

    BASH sudo yum install git

  • Fedora

    BASH sudo dnf install git

⚙️ 基础配置

安装完成后，你需要告诉 Git 你是谁（提交代码时会显示）：

BASH # 设置用户名
git config --global user.name "你的名字"

# 设置邮箱
git config --global user.email "your@email.com"

# 设置默认编辑器（可选，推荐 VS Code）
git config --global core.editor "code --wait"

# 查看配置
git config --list

💡 配置说明

  • --global 表示全局配置，对这台电脑所有仓库都生效
  • 邮箱建议使用 GitHub/GitLab 注册时用的邮箱
  • 配置信息保存在 ~/.gitconfig 文件中

📚 核心概念图解

理解这几个核心概念，你就掌握了 Git 的 80%：

📁 工作区（Working Directory）

你电脑里能看到的实际文件夹，你在里面增删改代码

📦 暂存区（Stage/Index）

临时存放修改的地方，准备提交的代码先放在这里

🗄️ 本地仓库（Repository）

Git 存储代码历史的地方，在 .git 隐藏文件夹里

☁️ 远程仓库（Remote）

GitHub/GitLab 上的仓库，用于多人协作和备份

代码流转过程

工作区修改 → git add
（放入暂存区） → git commit
（提交到本地仓库） → git push
（推送到远程仓库）

⚠️ 重要提醒

  • git add 只是把修改放入暂存区，随时可以撤回
  • git commit 才会真正保存版本，一定要写清楚提交信息
  • git push 才会把代码发到远程仓库，别人才能看到

⌨️ 常用命令详解

1. 仓库操作

BASH # 在当前文件夹新建 Git 仓库
git init

# 克隆远程仓库到本地
git clone https://github.com/用户名/仓库名.git

# 查看仓库状态
git status

2. 提交代码

BASH # 添加指定文件到暂存区
git add 文件名.js

# 添加所有修改到暂存区
git add .

# 提交暂存区的代码到本地仓库
git commit -m "修复登录bug"

# 添加并提交（一步完成，仅限已跟踪文件）
git commit -am "更新文档"

💡 提交信息规范

  • 用简洁的语言描述做了什么修改
  • 使用祈使句："修复" 而不是 "修复了"
  • 可以加上类型前缀：feat:新功能 fix:修复 docs:文档

3. 查看历史

BASH # 查看提交历史
git log

# 简洁版历史（一行显示）
git log --oneline

# 图形化显示分支历史
git log --graph --oneline

# 查看工作区与暂存区的差异
git diff

# 查看具体某次提交的修改
git show 提交ID

4. 撤销操作

BASH # 撤销工作区的修改（危险！修改会丢失）
git checkout -- 文件名.js

# 撤销暂存区的文件（取消 add）
git reset HEAD 文件名.js

# 修改最后一次提交
git commit --amend -m "新的提交信息"

# 回退到指定版本（保留修改）
git reset --soft HEAD~1

# 回退到指定版本（丢弃修改）
git reset --hard 提交ID

⚠️ 撤销操作要谨慎

  • --hard 会永久删除未提交的修改，使用前请确认
  • 已经 push 到远程的代码，不要直接回退，应该用 revert

🌿 分支管理

分支是 Git 最强大的功能之一，它让你可以在不影响主线的情况下开发新功能。

分支命令

BASH # 查看所有分支
git branch

# 创建新分支
git branch 新分支名

# 切换到指定分支
git checkout 分支名
# 或者更简洁的方式（创建并切换）
git checkout -b 新分支名

# 合并分支（把 feature 分支合并到当前分支）
git merge feature

# 删除已合并的分支
git branch -d 分支名

# 强制删除分支（未合并的）
git branch -D 分支名

推荐的分支策略（Git Flow 简化版）

main
（稳定版本） ← dev
（开发分支） ← feature/xxx
（功能分支）

💡 分支命名规范

  • feature/login - 新功能分支
  • bugfix/header-style - 修复 bug 分支
  • hotfix/critical - 紧急修复分支
  • docs/readme - 文档更新分支

分支操作示例

BASH # 1. 从 main 创建功能分支
git checkout main
git pull # 确保是最新的
git checkout -b feature/user-profile

# 2. 开发功能，提交代码
git add .
git commit -m "feat: 添加用户个人页面"

# 3. 功能完成，合并回 main
git checkout main
git merge feature/user-profile
git push origin main

# 4. 删除功能分支
git branch -d feature/user-profile

☁️ 远程仓库操作

配置远程仓库

BASH # 查看远程仓库地址
git remote -v

# 添加远程仓库
git remote add origin https://github.com/用户名/仓库名.git

# 修改远程仓库地址
git remote set-url origin https://新的地址.git

# 删除远程仓库关联
git remote remove origin

推送与拉取

BASH # 推送到远程仓库
git push origin 分支名
# 简写（如果已设置上游分支）
git push

# 拉取远程更新（不合并）
git fetch

# 拉取并合并远程更新
git pull
# 相当于 git fetch + git merge

# 第一次推送新分支
git push -u origin 分支名

GitHub 配置 SSH（推荐）

使用 HTTPS 每次都要输入密码，配置 SSH 后可以免密操作：

  • 生成 SSH 密钥

    ssh-keygen -t ed25519 -C "your@email.com"

    一路回车，使用默认设置

  • 复制公钥

    # Mac/Linux
    cat ~/.ssh/id_ed25519.pub

    # Windows
    type %userprofile%\.ssh\id_ed25519.pub

  • 添加到 GitHub

    登录 GitHub → Settings → SSH and GPG keys → New SSH key → 粘贴刚才的公钥

  • 测试连接

    ssh -T git@github.com

💡 团队协作建议

  • 每天开始工作前先 git pull 获取最新代码
  • 推送前先 pull，减少冲突
  • 不要直接在 main 分支上开发
  • 提交前检查自己修改了什么（git diff）

❓ 常见问题与解决

提交时提示 "Please tell me who you are" ▼

需要配置用户名和邮箱：

git config --global user.name "你的名字"
git config --global user.email "your@email.com"
push 时提示 "rejected: non-fast-forward" ▼

远程仓库有更新，你需要先 pull 再 push：

git pull origin main
git push origin main
pull 时提示 "CONFLICT"（冲突） ▼

你和别人改了同一处代码，需要手动解决冲突：

  1. 打开冲突文件，搜索 <<<<<<< 标记
  2. 决定保留哪部分代码，删除标记行
  3. 保存文件，然后 add 和 commit
git add .
git commit -m "解决冲突"
忘记添加文件到 .gitignore 已经提交了怎么办？ ▼
echo "文件名" >> .gitignore
git rm --cached 文件名
git commit -m "移除不该跟踪的文件"
误删了文件怎么恢复？ ▼

如果还没提交：

git checkout -- 文件名

如果已经提交了，查看历史找回：

git log --diff-filter=D --summary # 找到删除文件的提交
git checkout 提交ID^ -- 文件名
commit 信息写错了怎么改？ ▼

如果还没 push：

git commit --amend -m "新的提交信息"

如果已经 push 了，谨慎修改（会改变提交历史）：

git commit --amend -m "新的提交信息"
git push --force-with-lease
