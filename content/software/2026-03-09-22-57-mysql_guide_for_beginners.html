📖 第一章：什么是 MySQL

1.1 数据库是什么

想象你要管理一个图书馆，有成千上万本书。如果没有分类系统，找到一本书会非常困难。数据库就像是一个超级电子表格，帮助你：

  • 📦 存储数据 - 把信息有条理地保存起来
  • 🔍 查找数据 - 快速找到需要的信息
  • ✏️ 修改数据 - 方便地更新和删除
  • 📊 分析数据 - 统计和汇总信息

🌰 生活实例

你手机里的通讯录就是一个简单的数据库：

  • 每个人的信息（姓名、电话、地址）就是一条记录
  • 姓名、电话这些类别就是字段/列
  • 所有人的信息合起来就是一张表

1.2 为什么选择 MySQL

🆓 完全免费

开源软件，个人学习和商业使用都不要钱

🌍 最流行的数据库

⚡ 速度快

性能优秀，能处理大量数据和高并发访问

🔒 安全可靠

支持数据加密、用户权限管理等安全功能

支持 Windows、Mac、Linux 各种系统

⚙️ 第二章：安装与配置

2.1 Windows 安装

  1. 下载安装包

    访问 MySQL 官方下载页面，下载 mysql-installer-community（社区版，免费）

    💡 提示 选择 "Windows (x86, 32-bit), MSI Installer" 版本，大约 300MB
  2. 运行安装程序

    双击下载的 .msi 文件，选择安装类型：

    • Server only - 只安装数据库服务器（推荐）
    • Full - 安装所有组件
    • Custom - 自定义安装
  3. 配置 root 密码

    安装过程中会提示设置 root（管理员）密码，这个密码非常重要，一定要记住！

    ⚠️ 重要提醒 root 密码是数据库的最高权限密码，建议设置复杂密码并妥善保存。如果忘记会很麻烦！
  4. 完成安装

    一路点击 "Next" 或 "Execute"，直到安装完成。默认会开机自启动。

2.2 Mac 安装（推荐用 Homebrew）

# 1. 安装 Homebrew（如果没有） /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)" # 2. 安装 MySQL brew install mysql # 3. 启动 MySQL 服务 brew services start mysql # 4. 设置 root 密码 mysql_secure_installation

2.3 验证安装是否成功

打开命令行（Windows 按 Win + R，输入 cmd）：

# 登录 MySQL（-u 是用户名，-p 表示需要密码） mysql -u root -p # 输入密码后，看到以下提示表示成功： Welcome to the MySQL monitor. Commands end with ; or \g. Your MySQL connection id is 8 Server version: 8.0.xx mysql> # 看到这个 mysql> 提示符，恭喜你，安装成功！
💡 常用命令
  • exit 或 quit - 退出 MySQL
  • status - 查看当前状态
  • \q - 快捷退出

🎯 第三章：基础操作 - 创建你的第一个数据库

3.1 数据库操作

📁 创建数据库

-- 创建一个名为 school 的数据库 CREATE DATABASE school; -- 创建时指定字符集（推荐，避免中文乱码） CREATE DATABASE school CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

📂 查看和选择数据库

-- 查看所有数据库 SHOW DATABASES; -- 选择使用哪个数据库（重要！要先选数据库才能操作表） USE school; -- 查看当前使用的数据库 SELECT DATABASE(); -- 删除数据库（危险操作！） DROP DATABASE school;
⚠️ 警告 DROP DATABASE 会永久删除整个数据库及其所有数据，执行前务必确认！

3.2 创建数据表

以学生信息表为例，学习如何创建表：

CREATE TABLE students ( id INT PRIMARY KEY AUTO_INCREMENT, -- 学号，主键，自动增长 name VARCHAR(50) NOT NULL, -- 姓名，最多50字符，不能为空 age INT, -- 年龄 gender ENUM('男', '女'), -- 性别，只能是男或女 phone VARCHAR(20), -- 电话 email VARCHAR(100) UNIQUE, -- 邮箱，必须唯一 enrollment_date DATE, -- 入学日期 score DECIMAL(5,2) DEFAULT 0.00 -- 成绩，默认0，5位数字2位小数 );

🔍 数据类型速查表

类型 说明 示例
INT 整数 age INT → 18
VARCHAR(n) 可变长度字符串 name VARCHAR(50) → "张三"
TEXT 长文本 description TEXT
DECIMAL(m,d) 精确小数（m总位数，d小数位） price DECIMAL(10,2) → 99.99
DATE 日期（年月日） birthday DATE → '2000-05-20'
DATETIME 日期时间 created DATETIME → '2024-01-15 09:30:00'
BOOLEAN 布尔值（0或1） is_active BOOLEAN → 1
ENUM 枚举（限定选项） status ENUM('待处理','已完成')

📋 约束条件说明

🔑 PRIMARY KEY

主键，唯一标识每条记录，不能重复、不能为空

🚫 NOT NULL

非空约束，该字段必须有值

✨ UNIQUE

唯一约束，该字段的值不能重复

📎 DEFAULT

默认值，不填时自动填入

🔄 AUTO_INCREMENT

自动增长，每次+1，常用于ID

🔗 FOREIGN KEY

外键，关联其他表的字段

3.3 查看和修改表

-- 查看当前数据库所有表 SHOW TABLES; -- 查看表结构 DESCRIBE students; -- 或者简写 DESC students; -- 添加新列 ALTER TABLE students ADD COLUMN address VARCHAR(200); -- 修改列类型 ALTER TABLE students MODIFY COLUMN age TINYINT UNSIGNED; -- 删除列 ALTER TABLE students DROP COLUMN address; -- 重命名表 RENAME TABLE students TO pupils;

3.4 插入数据

-- 插入完整数据（按列顺序） INSERT INTO students VALUES (NULL, '张三', 20, '男', '13800138001', 'zhangsan@qq.com', '2023-09-01', 85.50); -- 推荐：指定列插入（更清晰、可跳过可为空的列） INSERT INTO students (name, age, gender, phone, enrollment_date) VALUES ('李四', 19, '女', '13900139002', '2023-09-01'); -- 批量插入（多条数据） INSERT INTO students (name, age, gender, phone, enrollment_date) VALUES ('王五', 21, '男', '13700137003', '2023-09-01'), ('赵六', 20, '女', '13600136004', '2023-09-01'), ('孙七', 19, '男', '13500135005', '2023-09-01');

3.5 更新和删除数据

-- 更新数据（记得加 WHERE 条件！） UPDATE students SET score = 90.00 WHERE id = 1; -- 更新多个字段 UPDATE students SET age = 21, score = 88.50 WHERE name = '李四'; -- 删除数据（极其危险，务必确认条件！） DELETE FROM students WHERE id = 5; -- ❌ 危险！不加 WHERE 会删除所有数据！ -- DELETE FROM students;
🚨 保命法则

执行 UPDATE 或 DELETE 前，先用 SELECT 确认影响范围！

SELECT * FROM students WHERE id = 5; ← 确认后再执行 DELETE

🔍 第四章：数据查询 - SELECT 的艺术

查询是数据库最常用的操作，SELECT 语句是 SQL 的核心。让我们从简单到复杂逐步学习。

4.1 基础查询

-- 查询所有学生的所有信息 SELECT * FROM students; -- 查询指定列 SELECT name, age, score FROM students; -- 给列起别名（更友好的显示） SELECT name AS 姓名, age AS 年龄, score AS 成绩 FROM students; -- 去重查询（看看有哪些不同的年龄） SELECT DISTINCT age FROM students;

4.2 条件查询 WHERE

-- 查询所有女生 SELECT * FROM students WHERE gender = '女'; -- 查询成绩大于 80 分的学生 SELECT * FROM students WHERE score > 80; -- 查询年龄在 18 到 22 之间的学生 SELECT * FROM students WHERE age BETWEEN 18 AND 22; -- 查询特定几个学生 SELECT * FROM students WHERE name IN ('张三', '李四', '王五'); -- 模糊查询：查询姓张的学生（% 表示任意字符） SELECT * FROM students WHERE name LIKE '张%'; -- 查询名字是两个字的学生（_ 表示单个字符） SELECT * FROM students WHERE name LIKE '__'; -- 组合条件：女生且成绩大于 80 SELECT * FROM students WHERE gender = '女' AND score > 80; -- 男生或年龄大于 20 SELECT * FROM students WHERE gender = '男' OR age > 20;

🔧 WHERE 条件运算符

运算符 含义 示例
= 等于 age = 20
!= 或 <> 不等于 gender != '男'
> 大于 score > 80
< 小于 age < 25
>= 大于等于 score >= 60
BETWEEN 在范围内 age BETWEEN 18 AND 22
LIKE 模糊匹配 name LIKE '张%'
IN 在列表中 id IN (1, 2, 3)
IS NULL 为空 email IS NULL

4.3 排序和分页

-- 按成绩升序排列（ASC 可省略） SELECT * FROM students ORDER BY score ASC; -- 按成绩降序排列 SELECT * FROM students ORDER BY score DESC; -- 先按性别排序，再按成绩排序 SELECT * FROM students ORDER BY gender ASC, score DESC; -- 分页查询：只显示前 5 条 SELECT * FROM students LIMIT 5; -- 分页查询：从第 6 条开始，显示 5 条（第 2 页） SELECT * FROM students LIMIT 5 OFFSET 5; -- MySQL 简写形式 SELECT * FROM students LIMIT 5, 5;
💡 分页公式

第 n 页，每页 pageSize 条：LIMIT (n-1) * pageSize, pageSize

示例：每页10条，第3页 → LIMIT 20, 10

4.4 聚合查询

-- 统计学生总数 SELECT COUNT(*) FROM students; -- 查询最高分 SELECT MAX(score) FROM students; -- 查询最低分 SELECT MIN(score) FROM students; -- 计算平均分 SELECT AVG(score) FROM students; -- 计算总成绩 SELECT SUM(score) FROM students; -- 综合统计 SELECT COUNT(*) AS 总人数, MAX(score) AS 最高分, MIN(score) AS 最低分, ROUND(AVG(score), 2) AS 平均分 FROM students;

4.5 分组查询 GROUP BY

-- 按性别统计人数 SELECT gender, COUNT(*) AS 人数 FROM students GROUP BY gender; -- 按性别统计平均分 SELECT gender AS 性别, COUNT(*) AS 人数, ROUND(AVG(score), 2) AS 平均分, MAX(score) AS 最高分 FROM students GROUP BY gender; -- HAVING 筛选分组后的结果（不能用 WHERE） SELECT gender, AVG(score) AS 平均分 FROM students GROUP BY gender HAVING 平均分 > 80;
⚠️ WHERE vs HAVING
  • WHERE - 在分组前过滤原始数据
  • HAVING - 在分组后过滤聚合结果

🔗 第五章：多表关联查询

实际应用中，数据通常分散在多个表中。比如学校系统可能有学生表、班级表、课程表等。

5.1 准备示例数据

-- 创建班级表 CREATE TABLE classes ( id INT PRIMARY KEY AUTO_INCREMENT, class_name VARCHAR(50) NOT NULL, teacher VARCHAR(50) ); -- 插入班级数据 INSERT INTO classes (class_name, teacher) VALUES ('计算机一班', '王老师'), ('计算机二班', '李老师'); -- 修改学生表，添加班级ID字段（如果不存在的话） ALTER TABLE students ADD COLUMN class_id INT; -- 更新学生班级 UPDATE students SET class_id = 1 WHERE id <= 3; UPDATE students SET class_id = 2 WHERE id > 3;

5.2 内连接 INNER JOIN

只返回两个表中匹配的记录：

-- 查询学生及其班级信息 SELECT s.name AS 姓名, s.age AS 年龄, c.class_name AS 班级, c.teacher AS 班主任 FROM students s INNER JOIN classes c ON s.class_id = c.id; -- 简写形式（效果相同） SELECT s.name, c.class_name FROM students s, classes c WHERE s.class_id = c.id;

5.3 左连接 LEFT JOIN

返回左表所有记录，右表不匹配的地方显示 NULL：

-- 查询所有学生（包括没有分配班级的） SELECT s.name AS 姓名, c.class_name AS 班级 FROM students s LEFT JOIN classes c ON s.class_id = c.id;

🌰 连接类型对比

  • INNER JOIN - 两个表都有匹配才显示（交集）
  • LEFT JOIN - 左表全显示，右表不匹配填 NULL
  • RIGHT JOIN - 右表全显示，左表不匹配填 NULL
  • FULL JOIN - 两边都显示，不匹配填 NULL（MySQL 不支持，可用 UNION）

🚀 第六章：进阶技巧与最佳实践

6.1 索引 - 让查询飞起来

索引就像书的目录，能快速定位数据，大幅提高查询速度。

-- 创建索引 CREATE INDEX idx_name ON students(name); -- 创建唯一索引 CREATE UNIQUE INDEX idx_email ON students(email); -- 复合索引（多列） CREATE INDEX idx_class_score ON students(class_id, score); -- 查看表的所有索引 SHOW INDEX FROM students; -- 删除索引 DROP INDEX idx_name ON students;
⚠️ 索引注意事项
  • 索引不是越多越好，太多会降低写入速度
  • 经常用于查询条件的字段适合加索引
  • 数据量小的表没必要加索引
  • 主键自动有索引

6.2 事务 - 保证数据安全

事务确保一系列操作要么全部成功，要么全部失败，不会出现中间状态。

-- 开启事务 START TRANSACTION; -- 或者 BEGIN; -- 执行一系列操作 UPDATE accounts SET balance = balance - 1000 WHERE id = 1; UPDATE accounts SET balance = balance + 1000 WHERE id = 2; -- 如果一切正常，提交事务 COMMIT; -- 如果出错，回滚事务（撤销所有操作） ROLLBACK;

6.3 备份与恢复

# ========== 命令行操作 ========== # 备份数据库（导出） mysqldump -u root -p school > school_backup.sql # 备份多个数据库 mysqldump -u root -p --databases school test > backup.sql # 备份所有数据库 mysqldump -u root -p --all-databases > all_databases.sql # 恢复数据库（导入） mysql -u root -p school < school_backup.sql # 在 MySQL 命令行内导入 SOURCE /path/to/school_backup.sql;

6.4 用户与权限管理

-- 创建新用户 CREATE USER 'xiaoming'@'localhost' IDENTIFIED BY 'password123'; -- 授予所有权限 GRANT ALL PRIVILEGES ON school.* TO 'xiaoming'@'localhost'; -- 只授予查询权限 GRANT SELECT ON school.* TO 'xiaoming'@'localhost'; -- 刷新权限（立即生效） FLUSH PRIVILEGES; -- 查看用户权限 SHOW GRANTS FOR 'xiaoming'@'localhost'; -- 撤销权限 REVOKE ALL PRIVILEGES ON school.* FROM 'xiaoming'@'localhost'; -- 删除用户 DROP USER 'xiaoming'@'localhost';

❓ 第七章：常见问题与解决

问题 解决方法
中文显示乱码 创建数据库时指定 CHARACTER SET utf8mb4；连接时执行 SET NAMES utf8mb4;
忘记 root 密码 跳过权限启动： mysqld --skip-grant-tables，然后更新 mysql.user 表
Can't connect 错误 检查服务是否启动；检查端口（默认3306）是否被占用；检查防火墙设置
Access denied 错误 用户名/密码错误；或该用户没有从当前主机连接的权限
查询太慢 添加索引；优化 SQL 语句；使用 EXPLAIN 分析执行计划
Table doesn't exist 检查表名拼写；确认 USE 了正确的数据库；Linux 下表名区分大小写
Duplicate entry 错误 插入的数据违反了唯一约束（UNIQUE 或 PRIMARY KEY）
