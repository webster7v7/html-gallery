1. 什么是 Spring Boot？

💡 一句话理解

Spring Boot 是一个让你快速、简单地搭建 Java Web 项目的框架。它帮你做好了各种配置，你只需要专注写业务代码！

为什么要学 Spring Boot？

⚡ 开箱即用

不用写一堆配置文件，几分钟就能跑起来一个项目

🔧 自动配置

根据你添加的依赖，自动完成相关配置

🏢 企业主流

国内 90% 以上的 Java 公司都在使用 Spring Boot

📦 内嵌服务器

内置 Tomcat，直接运行，不用额外部署

Spring Boot vs 传统 Spring

对比项 传统 Spring Spring Boot
配置方式 大量 XML 配置 注解 + 少量配置
项目搭建 步骤繁琐 一键生成
依赖管理 容易版本冲突 自动版本管理
部署方式 打 war 包放 Tomcat 直接运行 jar 包

2. 环境准备

搭建 Spring Boot 开发环境，只需要安装两样东西：

2.1 安装 JDK

📝 版本建议

推荐安装 JDK 17 或 JDK 21（长期支持版本），Spring Boot 3.x 要求最低 JDK 17

  • 下载 JDK

    访问 adoptium.net 下载开源版本，或 Oracle 官网下载

  • 安装

    双击安装包，一路点击"下一步"即可

  • 验证安装

    打开命令提示符（Win+R 输入 cmd），执行：

    java -version

    如果显示版本号，说明安装成功！

2.2 安装 IntelliJ IDEA

IDEA 是目前最流行的 Java 开发工具，有免费的社区版（Community）。

  • 下载 IDEA

    访问 jetbrains.com 下载 Community 版（免费）

  • 安装

    双击安装，建议勾选"Add to PATH"

  • 安装插件（可选）

    打开 IDEA → File → Settings → Plugins，搜索安装：

    • Chinese (Simplified) Language Pack（中文语言包）
    • Spring Boot Helper

3. 创建第一个 Spring Boot 项目

方法一：使用 Spring Initializr 官网（推荐新手）

  • 访问官网

    打开 start.spring.io

  • 配置项目信息
    Project Maven
    Language Java
    Spring Boot 3.2.x（最新稳定版）
    Group com.example（通常填公司域名倒写）
    Artifact demo（项目名称）
    Java 17 或 21
  • 添加依赖

    点击 "Add Dependencies"，选择：

    • Spring Web - 用于构建 Web 应用
    • Spring Boot DevTools - 热部署工具（开发时自动重启）
  • 生成项目

    点击 "Generate" 下载压缩包，解压后用 IDEA 打开

方法二：使用 IDEA 直接创建

File → New → Project → Spring Initializr
→ 选择 Server URL: https://start.spring.io
→ 填写项目信息 → 选择依赖 → 完成
⚠️ 注意

第一次打开项目时，IDEA 会自动下载依赖，可能需要几分钟时间，请耐心等待！

4. 编写 Hello World 接口

项目结构说明

demo/
├── src/
│   ├── main/
│   │   ├── java/
│   │   │   └── com/example/demo/
│   │   │       ├── DemoApplication.java    ← 启动类（程序入口）
│   │   │       └── controller/             ← 控制器文件夹
│   │   └── resources/
│   │       ├── static/                     ← 静态资源
│   │       ├── templates/                  ← 模板文件
│   │       └── application.properties      ← 配置文件
│   └── test/                               ← 测试代码
└── pom.xml                                 ← Maven 配置

第一步：创建 Controller

在 com.example.demo 包下新建 controller 文件夹，然后创建 HelloController.java：

package com.example.demo.controller;

import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class HelloController {

    @GetMapping("/hello")
    public String hello() {
        return "Hello, Spring Boot! 🌱";
    }
}
📝 代码解释
  • @RestController - 标记这是一个 REST 接口控制器
  • @GetMapping("/hello") - 当访问 /hello 路径时，执行下面的方法
  • 方法返回的字符串会作为响应内容返回给浏览器

第二步：运行项目

找到 DemoApplication.java（带 main 方法的类），点击左侧绿色小箭头 → Run

或在终端执行：

./mvnw spring-boot:run

第三步：测试接口

打开浏览器，访问：

http://localhost:8080/hello

看到 Hello, Spring Boot! 🌱 就说明成功了！🎉

小练习：带参数的接口

试试添加一个新接口，接收名字参数：

@GetMapping("/hello/name")
public String helloName(@RequestParam String name) {
    return "你好，" + name + "！欢迎学习 Spring Boot！";
}

访问：http://localhost:8080/hello/name?name=张三

5. 核心概念讲解

5.1 常用注解一览

注解 作用 使用位置
@SpringBootApplication 标记为 Spring Boot 启动类 主类
@RestController 声明 REST 控制器 类
@RequestMapping 映射请求路径 类/方法
@GetMapping 处理 GET 请求 方法
@PostMapping 处理 POST 请求 方法
@RequestParam 获取 URL 参数 参数
@PathVariable 获取路径变量 参数
@RequestBody 获取请求体 JSON 参数
@Autowired 自动注入依赖 字段/构造器
@Service 声明业务层组件 类
@Repository 声明数据访问层 类
@Component 通用组件标记 类

5.2 三层架构

Spring Boot 项目通常按以下分层：

🎮 Controller 层

接收请求、返回响应，处理参数校验。相当于"前台接待"

⚙️ Service 层

业务逻辑处理，如计算、判断、调用多个 DAO。相当于"业务经理"

💾 Mapper/DAO 层

数据库操作，增删改查。相当于"仓库管理员"

5.3 完整示例：用户查询

// 1. 实体类
public class User {
    private Long id;
    private String name;
    private Integer age;
    // getter 和 setter 省略...
}

// 2. Controller 层
@RestController
@RequestMapping("/users")
public class UserController {
    
    @Autowired
    private UserService userService;
    
    @GetMapping("/{id}")
    public User getUser(@PathVariable Long id) {
        return userService.getById(id);
    }
    
    @GetMapping
    public List<User> listUsers() {
        return userService.list();
    }
    
    @PostMapping
    public String addUser(@RequestBody User user) {
        userService.save(user);
        return "添加成功";
    }
}

// 3. Service 层
@Service
public class UserService {
    
    @Autowired
    private UserMapper userMapper;
    
    public User getById(Long id) {
        return userMapper.selectById(id);
    }
    
    public List<User> list() {
        return userMapper.selectList();
    }
    
    public void save(User user) {
        userMapper.insert(user);
    }
}

6. 连接 MySQL 数据库

6.1 添加依赖

在 pom.xml 中添加：

<dependencies>
    <!-- MySQL 驱动 -->
    <dependency>
        <groupId>com.mysql</groupId>
        <artifactId>mysql-connector-j</artifactId>
        <scope>runtime</scope>
    </dependency>
    
    <!-- MyBatis-Plus（简化数据库操作） -->
    <dependency>
        <groupId>com.baomidou</groupId>
        <artifactId>mybatis-plus-boot-starter</artifactId>
        <version>3.5.5</version>
    </dependency>
</dependencies>

6.2 配置数据库

修改 src/main/resources/application.properties：

# 数据库配置
spring.datasource.url=jdbc:mysql://localhost:3306/mydb?useUnicode=true&characterEncoding=utf-8
spring.datasource.username=root
spring.datasource.password=你的密码
spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver

# MyBatis-Plus 配置
mybatis-plus.configuration.log-impl=org.apache.ibatis.logging.stdout.StdOutImpl
mybatis-plus.global-config.db-config.logic-delete-field=deleted
mybatis-plus.global-config.db-config.logic-delete-value=1
mybatis-plus.global-config.db-config.logic-not-delete-value=0

6.3 创建实体类

@Data  // 需要添加 lombok 依赖
@TableName("user")
public class User {
    @TableId(type = IdType.AUTO)
    private Long id;
    private String name;
    private Integer age;
    private String email;
}

6.4 创建 Mapper

@Mapper
public interface UserMapper extends BaseMapper<User> {
    // 基础 CRUD 方法 BaseMapper 已提供
}

6.5 使用 Service

@Service
public class UserService extends ServiceImpl<UserMapper, User> {
    // 继承 ServiceImpl 获得大量现成方法
}
💡 MyBatis-Plus 是什么？

它是 MyBatis 的增强工具，提供了通用的 CRUD 方法，让你不用写 SQL 就能完成大部分数据库操作，大大提高开发效率！
