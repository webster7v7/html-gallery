❓ 什么是 Maven？

Maven 是 Apache 开源的 项目管理和构建工具，主要用于 Java 项目。它能帮你自动下载依赖、编译代码、运行测试、打包发布，让项目管理变得简单高效。

📦 依赖管理

自动下载和管理项目所需的 jar 包，不用手动找来找去

🔨 项目构建

编译、测试、打包、部署一键完成，标准化流程

📁 目录规范

约定优于配置，统一的目录结构，团队协作无障碍

🌐 中央仓库

全球共享的 jar 包仓库，海量依赖随取随用

💡 为什么要用 Maven？

  • 告别 jar 包地狱：不用手动下载和管理几十个 jar 包
  • 自动处理传递依赖：A 依赖 B，B 依赖 C，Maven 自动帮你下载 ABC
  • 项目结构标准化：任何人看到你的项目都知道代码在哪
  • 一键构建：一个命令完成编译、测试、打包全流程
  • IDE 无缝集成：IntelliJ IDEA、Eclipse 原生支持

⬇️ 安装与配置

检查 Java 环境

Maven 需要 JDK 才能运行，先检查是否已安装 Java：

Terminal
$ java -version
java version "17.0.8" 2023-07-18 LTS
Java(TM) SE Runtime Environment (build 17.0.8+9-LTS-211)

如果没有安装，先去 Oracle 官网或 Adoptium 下载安装 JDK。

下载安装 Maven

  • 官网下载

    访问 maven.apache.org/download.cgi，下载 Binary zip archive（Windows）或 tar.gz（Mac/Linux）

  • 解压到指定目录

    解压到 D:\apache-maven-3.9.x（Windows）或 /usr/local/apache-maven-3.9.x（Mac/Linux）

  • 配置环境变量

    Windows：系统属性 → 高级 → 环境变量 → Path → 添加 D:\apache-maven-3.9.x\bin

    Mac/Linux：编辑 ~/.bash_profile 或 ~/.zshrc：

    export MAVEN_HOME=/usr/local/apache-maven-3.9.x
    export PATH=$PATH:$MAVEN_HOME/bin

  • 验证安装

    验证
    $ mvn -v
    Apache Maven 3.9.4
    Maven home: D:\apache-maven-3.9.4
    Java version: 17.0.8
    ✓ 安装成功！

⚠️ 常见问题

  • 如果提示 "mvn 不是内部或外部命令"，检查 Path 配置是否正确
  • Mac/Linux 修改配置文件后，需要执行 source ~/.bash_profile 生效

📚 核心概念

POM（项目对象模型）

POM 是 Maven 的核心，每个 Maven 项目根目录下都有一个 pom.xml 文件，它是项目的"身份证"：

XML <project>
 <modelVersion>4.0.0</modelVersion>
 <groupId>com.example</groupId>
 <artifactId>demo</artifactId>
 <version>1.0-SNAPSHOT</version>
 <packaging>jar</packaging>
</project>

坐标（GAV）

Maven 使用三个坐标唯一确定一个项目或依赖：

groupId 组织/公司域名倒写，如 com.alibaba
artifactId 项目名称，如 druid、fastjson
version 版本号，如 1.2.3、2.0.0-SNAPSHOT

💡 版本号说明

  • 1.0.0 - 正式版（Release）
  • 1.0.0-SNAPSHOT - 开发版（快照，不稳定）
  • 1.0.0-RC1 - 候选发布版
  • 1.0.0-BETA - 测试版

仓库

仓库类型 说明 位置
本地仓库 本机存储下载的依赖 ~/.m2/repository
中央仓库 Maven 官方仓库 repo.maven.apache.org
远程仓库 公司私服或第三方仓库 Nexus/Artifactory

📁 标准项目结构

Maven 规定了统一的目录结构，约定优于配置：

my-project/                   # 项目根目录
src/
main/
java/                # 源代码
resources/          # 配置文件
test/
java/                # 测试代码
resources/          # 测试配置
pom.xml                 # 项目配置文件
README.md             # 项目说明
target/                    # 构建输出（编译后的类文件、jar包）

💡 目录说明

  • src/main/java - 放 Java 源代码
  • src/main/resources - 放配置文件（application.properties、XML 等）
  • src/test/java - 放单元测试代码（JUnit 等）
  • target - Maven 自动生成，存放编译结果，不要提交到 Git

⌨️ 常用命令

Maven 命令格式：mvn [插件]:[目标] 或 mvn [阶段]

mvn clean

清理 target 目录

mvn compile

编译源代码

mvn test

运行单元测试

mvn package

打包（jar/war）

mvn install

打包并安装到本地仓库

mvn deploy

部署到远程仓库

命令组合使用

常用组合
# 最常用：清理、编译、测试、打包
$ mvn clean package
# 跳过测试（快速打包）
$ mvn clean package -DskipTests
# 仅编译，不打包
$ mvn clean compile
# 安装到本地仓库供其他项目使用
$ mvn clean install

其他实用命令

命令 说明
mvn dependency:tree 查看依赖树
mvn dependency:analyze 分析依赖（找出未使用/缺失的依赖）
mvn help:effective-pom 查看最终生效的 pom 配置
mvn archetype:generate 创建新项目

📦 依赖管理

添加依赖

在 pom.xml 的 <dependencies> 标签中添加：

pom.xml <dependencies>
 <!-- Spring Boot Web -->
 <dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-web</artifactId>
  <version>3.2.0</version>
 </dependency>

 <!-- MySQL 驱动 -->
 <dependency>
  <groupId>mysql</groupId>
  <artifactId>mysql-connector-java</artifactId>
  <version>8.0.33</version>
 </dependency>
</dependencies>

依赖范围（scope）

scope 编译 测试 打包 典型场景
compile ✓ ✓ ✓ 默认，大部分依赖用这个
provided ✓ ✓ ✗ Servlet API（容器已提供）
runtime ✗ ✓ ✓ JDBC 驱动（编译时不需要）
test ✗ ✓ ✗ JUnit 等测试框架
system ✓ ✓ ✓ 本地 jar 包（不推荐）
pom.xml <dependency>
 <groupId>javax.servlet</groupId>
 <artifactId>javax.servlet-api</artifactId>
 <version>4.0.1</version>
 <scope>provided</scope>
</dependency>

依赖排除

排除某个依赖传递的依赖：

pom.xml <dependency>
 <groupId>com.example</groupId>
 <artifactId>some-library</artifactId>
 <version>1.0</version>
 <exclusions>
  <exclusion>
   <groupId>commons-logging</groupId>
   <artifactId>commons-logging</artifactId>
  </exclusion>
 </exclusions>
</dependency>

💡 找依赖坐标的技巧

  • Maven Central：search.maven.org
  • MvnRepository：mvnrepository.com（推荐，有使用统计）
  • IDE 辅助：IntelliJ IDEA 输入 groupId 自动提示 artifactId
  • Spring 项目：使用父 POM 管理版本，不用写 version

🔄 构建生命周期

Maven 有三套相互独立的生命周期：

clean - 清理生命周期

pre-clean → clean → post-clean

default - 默认构建生命周期（核心）

validate → compile → test → package → verify → install → deploy

site - 站点生成生命周期

pre-site → site → post-site → site-deploy

💡 生命周期执行规则

  • 执行某个阶段时，会自动执行该阶段之前的所有阶段
  • mvn package 会自动执行 validate → compile → test → package
  • 不同的生命周期可以组合：mvn clean package

🚀 实战演练

创建第一个 Maven 项目

  • 使用原型创建项目

    创建项目
    $ mvn archetype:generate \
     -DarchetypeGroupId=org.apache.maven.archetypes \
     -DarchetypeArtifactId=maven-archetype-quickstart \
     -DarchetypeVersion=1.4 \
     -DgroupId=com.example \
     -DartifactId=my-app \
     -Dversion=1.0-SNAPSHOT \
     -Dpackage=com.example

  • 进入项目目录

    $ cd my-app
    $ ls
    pom.xml src/

  • 编译项目

    $ mvn compile
    [INFO] BUILD SUCCESS

  • 运行测试

    $ mvn test
    Tests run: 1, Failures: 0, Errors: 0
    [INFO] BUILD SUCCESS

  • 打包项目

    $ mvn package
    [INFO] Building jar: target/my-app-1.0-SNAPSHOT.jar
    [INFO] BUILD SUCCESS

  • 运行程序

    $ java -cp target/my-app-1.0-SNAPSHOT.jar com.example.App
    Hello World!

IDEA 导入 Maven 项目

  1. 打开 IntelliJ IDEA
  2. File → Open，选择项目根目录（pom.xml 所在目录）
  3. IDEA 会自动识别 Maven 项目并下载依赖
  4. 等待右下角进度条完成（首次导入可能需要几分钟）
  5. 展开右侧 Maven 面板查看生命周期和插件

⚡ 配置国内镜像加速

默认从 Maven 中央仓库下载依赖较慢，配置阿里云镜像可以大幅提升下载速度：

配置方法

编辑 ~/.m2/settings.xml（没有则创建），添加镜像配置：

settings.xml <settings>
 <mirrors>
  <mirror>
   <id>aliyun</id>
   <name>阿里云公共仓库</name>
   <url>https://maven.aliyun.com/repository/public</url>
   <mirrorOf>central</mirrorOf>
  </mirror>
 </mirrors>
</settings>

网易 http://mirrors.163.com/maven/repository/maven-public/

💡 其他加速技巧

  • 公司内网通常有 Nexus 私服，优先使用公司仓库
  • IDEA 设置中开启 "Import Maven projects automatically"
  • 多人协作时，考虑使用统一的 settings.xml
