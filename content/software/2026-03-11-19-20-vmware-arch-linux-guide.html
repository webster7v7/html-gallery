🤔 为什么选择 VMware 安装 Arch？

对于零基础小白来说，在 VMware 虚拟机中安装 Arch Linux 是最安全、最推荐的方式！

VMware 的优势

优势 说明
🛡️ 零风险 完全独立于物理机，操作失误不会损坏真实系统
💾 快照功能 随时可以保存状态，搞砸了能一键恢复
🔄 易于重装 不满意可以直接删除重建，几分钟搞定
🖥️ 多系统并存 可以同时运行 Windows 和 Arch，随时切换
📸 方便截图录屏 便于记录学习过程、分享经验
🧪 随意折腾 可以放心尝试各种配置，不用担心系统崩溃
建议：先在 VMware 中熟练掌握 Arch，之后再考虑是否安装到物理机！

📋 准备工作

系统要求

  • Windows 10/11 或 Linux/macOS 主机系统
  • 至少 8GB 内存（推荐 16GB）
  • 至少 50GB 可用磁盘空间
  • CPU 支持虚拟化（Intel VT-x 或 AMD-V）
  • 稳定的网络连接

需要下载的软件

软件 下载地址 说明
VMware Workstation Pro Broadcom 官网 个人使用免费（需注册）
VMware Workstation Player Broadcom 官网 免费版，功能稍少
Arch Linux ISO archlinux.org 下载最新 ISO 镜像
注意：从 Broadcom 官网下载 VMware 需要注册账号。如果不想注册，也可以使用 VirtualBox（完全免费）。

开启 CPU 虚拟化

大部分电脑默认已开启，如果 VMware 提示虚拟化未启用，需要进入 BIOS 开启：

  1. 重启电脑，按 F2/Del/F12（根据主板品牌）进入 BIOS
  2. 找到 Intel VT-x 或 AMD-V / SVM Mode
  3. 设置为 Enabled
  4. 保存退出（通常是 F10）

⚙️ VMware 虚拟机配置

1安装 VMware

下载 VMware Workstation Pro 或 Player，双击安装包按提示安装即可。

安装提示：
  • Windows 安装时可能会提示重启，保存好工作再安装
  • 安装完成后需要重启电脑
  • Pro 版本需要许可证，个人使用可在 Broadcom 网站申请免费许可

2创建新虚拟机

打开 VMware，点击 "创建新的虚拟机"（Create a New Virtual Machine）

🖼️ 界面示意

VMware 主界面 → 点击 "创建新的虚拟机" 按钮

选择配置类型：

  • 典型（推荐） - 选择这个，简单快捷
  • 自定义 - 高级用户选择，可以详细配置硬件

3选择安装源

选择 "安装程序光盘映像文件(iso)"，然后浏览选择下载好的 Arch Linux ISO 文件。

🖼️ 界面示意

浏览按钮 → 找到下载的 archlinux-xxxx.xx.xx-x86_64.iso 文件

如果提示 "无法检测此光盘映像中的操作系统"，这是正常的，直接点"下一步"即可。

4选择客户机操作系统

客户机操作系统 Linux
版本 Other Linux 6.x kernel 64-bit（或最新的版本）

5命名虚拟机

虚拟机名称 Arch Linux（或你喜欢的名字）
位置 选择一个空间充足的磁盘（建议 SSD）
注意：确保选择的磁盘有足够的空间（至少 50GB 空闲）！

6指定磁盘容量

最大磁盘大小 64 GB 或更大（根据你的空间决定）
存储方式 将虚拟磁盘存储为单个文件（性能更好）
虚拟磁盘是按需分配的，不会立即占用 64GB 空间，用多少占多少。

7自定义硬件（重要！）

点击 "自定义硬件..." 按钮，进行以下配置：

内存配置

内存 4096 MB (4GB) 或更多

建议分配物理机内存的 1/3 到 1/2，但不超过 8192 MB（除非你要运行大型应用）

处理器配置

处理器数量 2 个 或更多
每个处理器的内核数量 2 个 或更多
虚拟化引擎 勾选 "虚拟化 Intel VT-x/EPT 或 AMD-V/RVI"（可选，用于嵌套虚拟化）

网络适配器

网络连接 NAT 模式（默认，最简单）或 桥接模式

显示器

3D 图形 可以取消勾选（提高兼容性）
配置完成后点击 "关闭" → "完成"，虚拟机创建成功！

💿 安装 Arch Linux

现在正式开始安装 Arch Linux！由于是虚拟机，不需要考虑分区会丢失数据的问题，放心操作！

1启动虚拟机

在 VMware 主界面选中 Arch Linux 虚拟机，点击 "开启此虚拟机"。

🖼️ 界面示意

选择 "Arch Linux install medium (x86_64, UEFI)" 进入安装环境

首次启动可能会提示安装 VMware Tools，先选择"以后提醒我"，等系统装完再处理。

2进入安装环境

看到 Arch Linux 启动菜单后，直接回车选择第一个选项，稍等片刻会进入命令行界面。

你应该看到类似这样的提示：

📟 终端
root@archiso ~ # _

这表示你已经进入了 Arch 的安装环境。

3检查网络连接

VMware 的 NAT 网络通常自动连接，验证一下：

📟 终端
ping archlinux.org -c 4

如果看到 64 bytes from... 的回复，说明网络正常！按 Ctrl+C 停止。

4更新系统时间

📟 终端
timedatectl set-ntp true
timedatectl status

5分区（虚拟机简单方案）

查看磁盘：

📟 终端
fdisk -l

你应该看到 /dev/sda 或 /dev/nvme0n1，这就是虚拟磁盘。

💡 VMware 虚拟机分区方案（推荐新手）

由于是虚拟机，不需要复杂的分区，简单分两个区即可：

  • EFI 分区：512MB（用于启动）
  • 根分区：剩余全部空间（系统和数据放在一起）

使用 cfdisk 分区（图形化界面，适合新手）：

📟 终端
cfdisk /dev/sda

操作步骤：

  1. 选择 gpt 分区表类型，回车
  2. 选择 [ New ] 新建分区
  3. 输入大小 512M，回车
  4. 类型选择 EFI System
  5. 再次选择 [ New ]
  6. 大小保持默认（全部剩余空间），回车
  7. 选择 [ Write ] 写入更改
  8. 输入 yes 确认
  9. 选择 [ Quit ] 退出

6格式化分区

📟 终端
# 格式化 EFI 分区为 FAT32
mkfs.fat -F32 /dev/sda1

# 格式化根分区为 ext4
mkfs.ext4 /dev/sda2

7挂载分区

📟 终端
# 挂载根分区
mount /dev/sda2 /mnt

# 创建并挂载 EFI 分区
mkdir -p /mnt/boot/EFI
mount /dev/sda1 /mnt/boot/EFI

# 检查挂载情况
lsblk

8安装基础系统

📟 终端
# 先替换国内镜像源（提速，避免下载慢）
cp /etc/pacman.d/mirrorlist /etc/pacman.d/mirrorlist.bak
echo 'Server = https://mirrors.ustc.edu.cn/archlinux/$repo/os/$arch' > /etc/pacman.d/mirrorlist
pacman -Syy  # 刷新源

# 安装基础系统
pacstrap /mnt base base-devel linux linux-firmware vim nano networkmanager grub efibootmgr

这会安装基础系统、内核、网络管理器和引导程序。根据网络速度，可能需要 5-15 分钟。

如果下载很慢，可以先配置国内镜像源（见下一节"安装后配置"）。

9生成 fstab

📟 终端
genfstab -U /mnt >> /mnt/etc/fstab

# 检查生成的文件
cat /mnt/etc/fstab

10进入新系统

📟 终端
arch-chroot /mnt

现在你已经"进入"了新安装的系统，后续的设置都会保存到这个系统中。

11设置时区

📟 终端
ln -sf /usr/share/zoneinfo/Asia/Shanghai /etc/localtime
hwclock --systohc

12设置语言

📟 终端
# 编辑 locale.gen
nano /etc/locale.gen

找到并取消注释（删除行首的 #）：

en_US.UTF-8 UTF-8
zh_CN.UTF-8 UTF-8

保存：Ctrl+O → 回车 → Ctrl+X

📟 终端
# 生成 locale
locale-gen

# 设置默认语言
echo "LANG=zh_CN.UTF-8" > /etc/locale.conf

13设置主机名

📟 终端
# 设置主机名（可自定义）
echo "arch-vm" > /etc/hostname

# 编辑 hosts
nano /etc/hosts

添加以下内容：

127.0.0.1   localhost
::1         localhost
127.0.1.1   arch-vm.localdomain   arch-vm

14设置 root 密码

📟 终端
passwd

输入两次密码（不会显示），请记住这个密码！

15安装 GRUB 引导

📟 终端
# 安装 GRUB 到 EFI 分区
grub-install --target=x86_64-efi --efi-directory=/boot/EFI --bootloader-id=GRUB

# 生成配置文件
grub-mkconfig -o /boot/grub/grub.cfg
如果显示 "Installation finished. No error reported." 说明安装成功！

16启用网络服务

📟 终端
systemctl enable NetworkManager

17退出并重启

📟 终端
# 退出 chroot 环境
exit

# 卸载分区
umount -R /mnt

# 重启
reboot
重启后，在 VMware 启动画面快速按 ESC 键，进入 Boot Manager 选择从硬盘启动，或者等待自动进入 GRUB。
🎉 恭喜！ Arch Linux 基础安装完成！现在你应该看到登录提示，输入 root 和刚才设置的密码即可登录。

🔧 安装后配置

1. 创建普通用户

使用 root 登录后，创建一个日常使用的普通用户：

📟 终端
# 创建用户（替换 yourname 为你想要的用户名）
useradd -m -G wheel yourname

# 设置密码
passwd yourname

# 安装 sudo
pacman -S sudo

# 赋予 sudo 权限
EDITOR=nano visudo

找到 # %wheel ALL=(ALL:ALL) ALL，删除行首的 #，保存退出。

2. 配置国内镜像源（加速下载）

📟 终端
# 安装 reflector
sudo pacman -S reflector

# 自动选择最快镜像
sudo reflector --country China --protocol https --latest 10 --sort rate --save /etc/pacman.d/mirrorlist

3. 添加 Arch Linux CN 源

📟 终端
sudo nano /etc/pacman.conf

在文件末尾添加：

[archlinuxcn]
Server = https://mirrors.tuna.tsinghua.edu.cn/archlinuxcn/$arch

然后导入密钥：

📟 终端
sudo pacman -Sy
sudo pacman -S archlinuxcn-keyring

4. 安装常用工具

📟 终端
# 基本工具
sudo pacman -S git wget curl neofetch htop

# 中文字体
sudo pacman -S noto-fonts-cjk wqy-microhei

# 安装 yay (AUR 助手)
sudo pacman -S yay

5. 安装桌面环境（可选）

如果想使用图形界面，安装 KDE Plasma（推荐新手）：

📟 终端
# 安装 KDE Plasma
sudo pacman -S plasma-meta kde-applications

# 安装显示管理器
sudo pacman -S sddm
sudo systemctl enable sddm

# 重启进入图形界面
reboot
重启后会自动进入登录界面（SDDM），选择你的用户登录即可进入 KDE 桌面。

🛠️ 安装 VMware Tools

VMware Tools 是 VMware 的增强工具，安装后可以：
  • 自适应调整虚拟机分辨率
  • 在主机和虚拟机之间复制粘贴
  • 共享文件夹
  • 更好的鼠标和显卡性能

安装 open-vm-tools（推荐）

Arch 官方仓库提供了开源的 VMware Tools：

📟 终端
# 安装 open-vm-tools
sudo pacman -S open-vm-tools

# 安装桌面环境集成（如果安装了桌面）
sudo pacman -S open-vm-tools-desktop

# 启用服务
sudo systemctl enable vmtoolsd
sudo systemctl start vmtoolsd

验证安装

📟 终端
# 检查服务状态
systemctl status vmtoolsd

# 查看 VMware Tools 版本
vmware-toolbox-cmd -v

设置共享文件夹（可选）

  1. 在 VMware 菜单中：虚拟机 → 设置 → 选项 → 共享文件夹
  2. 选择"总是启用"，添加要共享的主机文件夹
  3. 在虚拟机中挂载：
📟 终端
# 创建挂载点
mkdir -p ~/shared

# 挂载共享文件夹（替换 shared-folder-name 为你设置的名称）
sudo mount -t fuse.vmhgfs-fuse .host:/shared-folder-name ~/shared -o allow_other

# 要开机自动挂载，编辑 /etc/fstab
sudo nano /etc/fstab

添加：

.host:/shared-folder-name /home/yourname/shared fuse.vmhgfs-fuse allow_other,defaults 0 0
安装 VMware Tools 后，重启虚拟机，然后尝试调整 VMware 窗口大小，分辨率应该会自动适应！

📸 使用快照功能（救命神器）

快照是 VMware 最强大的功能之一！ 它可以将虚拟机当前状态保存下来，随时恢复。

什么时候创建快照？

  • ✅ 系统刚安装完成，配置好基础环境后
  • ✅ 安装大型软件或更新系统前
  • ✅ 尝试不确定的操作前
  • ✅ 系统运行正常，想保留这个好状态时

如何创建快照

  1. 关闭虚拟机（或挂起）
  2. VMware 菜单：虚拟机 → 快照 → 拍摄快照
  3. 输入快照名称和描述（如："刚安装完成，已配置好网络"）
  4. 点击"拍摄快照"
🖼️ 界面示意

虚拟机 → 快照 → 拍摄快照 → 填写名称 → 拍摄快照

如何恢复快照

如果系统搞砸了，轻松恢复：

  1. 关闭虚拟机
  2. VMware 菜单：虚拟机 → 快照 → 恢复到快照
  3. 选择要恢复的快照
  4. 点击"恢复"
注意：恢复快照会丢失快照之后做的所有更改！重要文件记得备份！

快照管理技巧

操作 路径
查看所有快照 虚拟机 → 快照 → 快照管理器
删除快照 快照管理器 → 选择快照 → 删除
克隆虚拟机 虚拟机 → 管理 → 克隆（基于快照创建新虚拟机）
建议：定期清理不需要的旧快照，因为快照会占用磁盘空间。

❓ 常见问题解决

1. 无法启动，卡在黑屏

可能原因：显卡兼容性问题

解决：编辑虚拟机设置 → 显示器 → 取消勾选"加速 3D 图形"

2. 网络无法连接

检查步骤：
  1. 确认 VMware 虚拟网络服务已启动（Windows 服务中）
  2. 虚拟机设置 → 网络适配器 → 确认选择 NAT 或桥接模式
  3. 在 Arch 中检查网络服务：systemctl status NetworkManager
  4. 如果未运行，启动它：sudo systemctl start NetworkManager

3. 分辨率无法调整

解决：
  1. 确保已安装 open-vm-tools-desktop
  2. 重启虚拟机
  3. VMware 菜单：查看 → 自动调整大小 → 自动适应客户机

4. 无法复制粘贴

解决：
  1. 确认已安装 open-vm-tools-desktop
  2. VMware 菜单：虚拟机 → 可移动设备 → 确保没有禁用复制粘贴
  3. 重启 vmtoolsd 服务：sudo systemctl restart vmtoolsd

5. 更新系统后无法启动

解决：
  1. 使用 Arch ISO 重新启动虚拟机
  2. 按照安装步骤中的挂载方式，挂载分区
  3. arch-chroot /mnt 进入系统
  4. 检查并修复问题（如重新安装内核、重建 initramfs 等）
  5. 或者：直接恢复快照！这就是用虚拟机的好处 😄

6. 磁盘空间不足

解决：
  1. 关闭虚拟机
  2. VMware 菜单：虚拟机 → 设置 → 硬盘 → 扩展
  3. 输入新的最大磁盘大小
  4. 启动虚拟机，使用 GParted 扩展分区

7. 想备份整个虚拟机

方法：
  1. 关闭虚拟机
  2. 找到虚拟机文件夹（创建时指定的位置）
  3. 复制整个文件夹到安全位置
  4. 需要恢复时，VMware → 文件 → 打开 → 选择 .vmx 文件
记住：在 VMware 里折腾 Arch，最坏的情况就是删除虚拟机重建！放心大胆地学习和尝试吧！

💡 VMware + Arch 实用技巧

快捷键速查

快捷键 作用
Ctrl+G 将输入定向到虚拟机（点击虚拟机内部）
Ctrl+Alt 将输入从虚拟机释放（回到主机）
Ctrl+Alt+Enter 全屏模式
Ctrl+Alt+Insert 向虚拟机发送 Ctrl+Alt+Delete
Ctrl+Shift+Alt+T 在 VMware 中打开终端

性能优化建议

  • 将虚拟机磁盘文件放在 SSD 上，大幅提升 IO 性能
  • 分配足够的内存，但不要超过物理机的一半
  • 如果不需要，可以关闭虚拟机的 3D 加速
  • 定期整理虚拟机磁盘：虚拟机 → 管理 → 清理磁盘
  • 禁用不必要的启动服务，加快启动速度

学习路线建议

  1. 第一阶段：熟悉命令行基本操作（ls, cd, cp, mv 等）
  2. 第二阶段：学习 pacman 包管理，安装常用软件
  3. 第三阶段：尝试安装和配置桌面环境
  4. 第四阶段：深入理解 Arch 的配置文件和系统结构
  5. 第五阶段：尝试在物理机上安装（可选）
🎉 祝你学习愉快！ Arch Linux 的学习曲线虽然陡峭，但收获绝对值得。记住：Wiki 是你的好朋友 → wiki.archlinux.org
