GitHub 零基础入门指南

不需要懂编程，手把手教你使用全球最大的代码托管平台，开启你的开源之旅

👉 立即注册 GitHub

🤔什么是 GitHub？一句话解释

💡 通俗理解：GitHub 就像是一个「网盘 + 时光机 + 协作工具」
  • 网盘：存储你的代码文件
  • 时光机：可以回到文件的任何一个历史版本
  • 协作工具：多人一起编辑，不会互相覆盖

想象一下你和同学一起写一份报告：

  • ❌ 传统方式：用微信传文件，最后不知道哪个是最新版
  • ✅ GitHub 方式：所有人都在同一个地方编辑，自动合并修改，能看到谁改了什么

📝第一步：注册账号（5分钟）

1

访问官网

打开 github.com，点击右上角的 Sign up

🖼️ 官网首页截图位置
点击 Sign up 开始注册
2

填写信息

按提示填写：

  • 邮箱：建议用常用邮箱（QQ邮箱也可以）
  • 密码：至少8位，包含字母和数字
  • 用户名：只能用英文、数字和短横线，会出现在你的链接里
⚠️ 注意：用户名一旦确定很难修改，建议用 你的名字拼音 或 英文名
3

验证邮箱

注册后会收到验证邮件，点击邮件里的链接完成验证。如果没收到，检查垃圾箱。

📚第二步：搞懂这4个核心概念

📁

仓库 Repository

简称 Repo，相当于一个项目文件夹。你的所有代码、文档、图片都放在这里。

💾

提交 Commit

保存当前文件状态，就像游戏的存档点。可以随时回退到任何一个提交。

🌿

分支 Branch

从主线分出来的副本，可以在分支上随意修改，不影响主线。改完再合并回去。

🔄

推送 Push / 拉取 Pull

Push：把本地文件上传到 GitHub
Pull：把 GitHub 上的文件下载到本地

🎯 记忆口诀：

"仓库是房子，提交是拍照，分支是平行宇宙，推送pull是收发快递"

🎉第三步：创建你的第一个仓库

1

点击 New Repository

登录后，点击右上角 + 号，选择 New repository

🖼️ 创建仓库按钮位置
2

填写仓库信息

  • Repository name：仓库名，用英文，比如 my-first-project
  • Description：描述（可选），比如"我的第一个项目"
  • Public/Private：
    • Public（公开）：所有人都能看到，免费 ✅
    • Private（私有）：只有你能看到，免费 ✅
  • Add a README file：☑️ 勾选上，会自动创建一个说明文件
🖼️ 填写仓库信息表单
3

创建完成！

点击 Create repository，你就拥有了自己的第一个仓库！🎊

现在你可以：

  • 点击 README.md 文件，然后点击 ✏️ 编辑按钮修改内容
  • 尝试写一些文字，比如"# 你好，GitHub！"
  • 滚动到页面底部，点击 Commit changes 保存

💻第四步：安装 Git 并连接（电脑操作）

1

下载安装 Git

访问 git-scm.com/downloads 下载对应系统的版本

  • Windows：下载 .exe 文件，一路下一步安装
  • Mac：下载 .dmg 文件安装，或在终端运行 git --version 自动安装
2

配置你的信息

打开终端（Windows 用 Git Bash 或 CMD，Mac 用 Terminal），输入：

Terminal 
# 配置用户名（替换为你的GitHub用户名）
git config --global user.name "你的用户名"

# 配置邮箱（替换为你注册GitHub的邮箱）
git config --global user.email "your@email.com"
3

克隆仓库到本地

在 GitHub 仓库页面点击 Code 按钮，复制 HTTPS 链接，然后在终端输入：

Terminal 
# 进入你想存放项目的文件夹（示例）
cd Desktop

# 克隆仓库（替换为你的仓库链接）
git clone https://github.com/你的用户名/仓库名.git

# 进入项目文件夹
cd 仓库名

📋 常用 Git 命令速查表

git status
查看当前状态，哪些文件被修改了
git add 文件名
将文件添加到暂存区（准备提交）
git add .
添加所有修改过的文件
git commit -m "描述"
提交更改，-m 后面写这次修改的说明
git push
将本地提交推送到 GitHub
git pull
从 GitHub 拉取最新代码
git log
查看提交历史记录
📝 标准工作流程： 
# 1. 修改文件...
# 2. 查看修改状态
git status

# 3. 添加修改的文件
git add .

# 4. 提交（拍照存档）
git commit -m "修复了登录bug"

# 5. 推送到 GitHub
git push

💻

1

Fork 项目

进入你想贡献的开源项目页面，点击右上角的 Fork 按钮。

这会在你的账号下创建一个该项目的副本，你可以随意修改而不会影响原项目。

2

Clone 你 Fork 的仓库到本地，修改代码，然后 push 到你的 GitHub。

3

发起 Pull Request

在你的 Fork 页面点击 Contribute → Open pull request

填写修改说明，提交后原作者就能收到你的贡献请求。如果审核通过，你的代码就会合并到原项目中！🎉

❓常见问题 FAQ

Q1

Git 和 GitHub 有什么区别？

Git 是一个软件，安装在你电脑上，用来管理代码版本。

GitHub 是一个网站，用来存放你的代码，让其他人可以看到和协作。

关系：Git 是工具，GitHub 是平台。

Q2

必须要会编程才能用 GitHub 吗？

完全不需要！GitHub 可以存任何文件：

  • 写文档、笔记（Markdown 格式）
  • 托管静态网页（GitHub Pages）
  • 管理图片、音乐等文件
  • 写小说、博客
Q3

GitHub 是免费的吗？

是的！个人账户完全免费，包括：

  • ✅ 无限个公开仓库
  • ✅ 无限个私有仓库
  • ✅ 无限个协作者
  • ❌ 付费功能主要是高级的团队管理工具
Q4

我误删了文件怎么办？

不用慌！Git 记录了所有历史版本：

# 查看历史记录
git log

# 恢复到某个版本（替换commit_id为具体的ID）
git checkout commit_id -- 文件名
