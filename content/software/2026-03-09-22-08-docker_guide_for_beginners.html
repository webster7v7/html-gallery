🤔 什么是 Docker？

Docker 是一个开源的容器化平台，它可以让开发者将应用程序及其依赖打包到一个轻量级、可移植的容器中，然后发布到任何流行的 Linux 或 Windows 机器上。

💡 通俗理解：想象你要搬家，传统方式是把家具一件件搬到新家再组装。Docker 就像是把整间屋子（连同家具）一起打包成一个"集装箱"，直接搬到新地方就能住！

为什么要用 Docker？

❌ 传统部署

  • "在我电脑上能运行啊"问题
  • 环境配置复杂，容易出错
  • 一台服务器一个应用
  • 资源利用率低
  • 迁移困难

✅ Docker 部署

  • 一次构建，到处运行
  • 环境一致性保证
  • 一台服务器多容器
  • 资源隔离、高效利用
  • 快速迁移扩展

Docker 的主要优势

  • 🚀 快速部署 - 秒级启动，分钟级部署
  • 📦 环境一致 - 开发、测试、生产环境完全一致
  • 💰 节省资源 - 一台机器运行多个隔离应用
  • 🔄 易于迁移 - 镜像可在任何 Docker 环境运行
  • 🛠️ 简化配置 - 将配置打包在镜像中

💻 安装 Docker

Windows / macOS 安装

  1. 下载 Docker Desktop

    访问 Docker 官网 下载对应系统的安装包

  2. 运行安装程序

    双击下载的安装包，按提示完成安装（Windows 需要开启 WSL2）

  3. 验证安装
    docker --version

Linux 安装（Ubuntu/Debian）

# 更新软件包索引 sudo apt update # 安装必要的依赖 sudo apt install apt-transport-https ca-certificates curl gnupg lsb-release # 添加 Docker 官方 GPG 密钥 curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg # 添加 Docker 软件源 echo "deb [arch=amd64 signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null # 安装 Docker sudo apt update sudo apt install docker-ce docker-ce-cli containerd.io # 将当前用户添加到 docker 组（免 sudo） sudo usermod -aG docker $USER # 验证安装 docker --version
✅ 安装验证：在终端输入 docker run hello-world，如果看到 "Hello from Docker!" 说明安装成功！

📚 核心概念

理解 Docker 的三个核心概念是学习 Docker 的关键：

📦

镜像 (Image)

只读模板，包含运行应用所需的一切：代码、运行时、库、环境变量等

🐳

容器 (Container)

镜像的运行实例，是独立、隔离的运行环境，可以被创建、启动、停止、删除

🏪

仓库 (Registry)

存储和分发镜像的地方，如 Docker Hub 是官方的公共仓库

💡 类比理解：
  • 镜像 = 类（Class）
  • 容器 = 对象（Object）
  • 仓库 = GitHub（代码托管平台）

📦 镜像操作详解

1. 搜索镜像

# 在 Docker Hub 搜索镜像 docker search nginx # 查看镜像详情 docker search --filter=stars=100 nginx

2. 拉取镜像

# 拉取最新版本 docker pull nginx # 拉取指定版本（推荐） docker pull nginx:1.24.0 # 拉取指定平台的镜像 docker pull --platform linux/amd64 nginx
💡 版本标签建议：生产环境务必指定版本标签（如 :1.24.0），不要用 latest，避免版本升级带来的意外问题！

3. 查看本地镜像

# 列出所有镜像 docker images # 只显示镜像 ID docker images -q # 查看镜像历史 docker history nginx

4. 删除镜像

# 删除指定镜像 docker rmi nginx # 强制删除（包括正在使用的） docker rmi -f nginx # 删除所有悬空镜像 docker image prune # 删除所有未使用的镜像 docker image prune -a

5. 保存和加载镜像

# 保存镜像为 tar 文件 docker save -o nginx-backup.tar nginx:latest # 从 tar 文件加载镜像 docker load -i nginx-backup.tar # 导出容器为 tar（仅文件系统） docker export -o container.tar 容器ID

🐳 容器操作详解

1. 创建并运行容器

# 前台运行（阻塞终端） docker run nginx # 后台运行（推荐） docker run -d nginx # 指定名称 docker run -d --name my-nginx nginx # 端口映射（主机端口:容器端口） docker run -d -p 8080:80 nginx # 挂载卷（主机目录:容器目录） docker run -d -v /host/data:/usr/share/nginx/html nginx # 自动重启 docker run -d --restart always nginx # 设置环境变量 docker run -d -e MYSQL_ROOT_PASSWORD=123456 mysql # 综合示例 docker run -d \\ --name my-web \\ -p 8080:80 \\ -v /home/user/html:/usr/share/nginx/html \\ --restart always \\ nginx:alpine
💡 常用参数说明：
  • -d - 后台运行（detached mode）
  • -p - 端口映射
  • -v - 卷挂载（持久化数据）
  • --name - 指定容器名称
  • --restart - 重启策略
  • -e - 环境变量

2. 查看容器

# 查看运行中的容器 docker ps # 查看所有容器（包括停止的） docker ps -a # 查看容器日志 docker logs 容器ID # 实时跟踪日志 docker logs -f 容器ID # 查看容器详细信息 docker inspect 容器ID

3. 管理容器

# 停止容器 docker stop 容器ID # 启动容器 docker start 容器ID # 重启容器 docker restart 容器ID # 暂停容器 docker pause 容器ID # 恢复容器 docker unpause 容器ID # 删除容器 docker rm 容器ID # 强制删除运行中的容器 docker rm -f 容器ID # 删除所有停止的容器 docker container prune

4. 进入容器

# 进入容器执行命令（推荐） docker exec -it 容器ID /bin/bash # 如果容器没有 bash，使用 sh docker exec -it 容器ID /bin/sh # 执行单次命令 docker exec 容器ID ls -la # 以 root 用户进入 docker exec -it -u root 容器ID /bin/bash
⚠️ 注意：使用 docker attach 进入容器后，如果 exit 会导致容器停止，建议使用 docker exec！

5. 复制文件

# 从主机复制到容器 docker cp ./local-file.txt 容器ID:/container/path/ # 从容器复制到主机 docker cp 容器ID:/container/file.txt ./local-path/

📝 Dockerfile 入门

Dockerfile 是用来构建 Docker 镜像的文本文件，包含一系列指令。

常用指令

指令 说明 示例
FROM 基础镜像 FROM node:18-alpine
WORKDIR 工作目录 WORKDIR /app
COPY 复制文件 COPY . .
RUN 执行命令 RUN npm install
ENV 环境变量 ENV NODE_ENV=production
EXPOSE 暴露端口 EXPOSE 3000
CMD 容器启动命令 CMD ["node", "app.js"]

Node.js 应用示例

📁 项目结构

my-node-app/ ├── Dockerfile ├── package.json ├── package-lock.json └── app.js
# 使用官方 Node.js 轻量级镜像 FROM node:18-alpine # 设置工作目录 WORKDIR /app # 复制 package 文件 COPY package*.json ./ # 安装依赖 RUN npm ci --only=production # 复制应用代码 COPY . . # 暴露端口 EXPOSE 3000 # 启动应用 CMD ["node", "app.js"]

构建镜像

# 构建镜像（注意最后的点） docker build -t my-node-app:1.0 . # 不使用缓存构建 docker build --no-cache -t my-node-app:1.0 . # 查看构建历史 docker history my-node-app:1.0

Python 应用示例

# 使用官方 Python 镜像 FROM python:3.11-slim # 设置工作目录 WORKDIR /app # 复制依赖文件 COPY requirements.txt . # 安装依赖 RUN pip install --no-cache-dir -r requirements.txt # 复制代码 COPY . . # 暴露端口 EXPOSE 5000 # 运行应用 CMD ["python", "app.py"]

🚀 实战案例

案例 1：部署 Nginx 静态网站

# 1. 创建本地目录存放网页 mkdir -p ~/docker-nginx/html cd ~/docker-nginx # 2. 创建简单的 HTML 文件 echo '

Hello Docker Nginx!

' > html/index.html # 3. 运行 Nginx 容器 docker run -d \\ --name my-nginx \\ -p 80:80 \\ -v $(pwd)/html:/usr/share/nginx/html \\ --restart always \\ nginx:alpine # 4. 访问 http://localhost 查看效果

案例 2：部署 MySQL 数据库

# 1. 创建数据卷目录 mkdir -p ~/docker-mysql/data # 2. 运行 MySQL 容器 docker run -d \\ --name my-mysql \\ -p 3306:3306 \\ -v ~/docker-mysql/data:/var/lib/mysql \\ -e MYSQL_ROOT_PASSWORD=your_password \\ -e MYSQL_DATABASE=mydb \\ --restart always \\ mysql:8.0 # 3. 连接 MySQL docker exec -it my-mysql mysql -uroot -p # 4. 或使用本地客户端连接 localhost:3306

案例 3：部署 Redis 缓存

# 运行 Redis 容器 docker run -d \\ --name my-redis \\ -p 6379:6379 \\ --restart always \\ redis:alpine # 连接 Redis CLI docker exec -it my-redis redis-cli # 测试 127.0.0.1:6379> set name docker 127.0.0.1:6379> get name

案例 4：完整的 Web 应用栈

# 启动后端 API（Node.js） docker run -d \\ --name api-server \\ -p 3000:3000 \\ -e NODE_ENV=production \\ -e DB_HOST=mysql \\ --link my-mysql:mysql \\ my-api:1.0 # 启动前端（Nginx） docker run -d \\ --name frontend \\ -p 80:80 \\ --link api-server:api \\ my-frontend:1.0

🎼 Docker Compose 多容器编排

Docker Compose 是用于定义和运行多容器 Docker 应用的工具，使用 YAML 文件配置应用服务。

安装 Docker Compose

# Docker Desktop 已自带 Compose # 验证安装 docker compose version # Linux 用户可能需要单独安装 sudo apt install docker-compose-plugin

docker-compose.yml 示例

version: '3.8' services: # Web 服务 web: image: nginx:alpine container_name: my-web ports: - "80:80" volumes: - ./html:/usr/share/nginx/html depends_on: - api restart: always # API 服务 api: build: ./api container_name: my-api ports: - "3000:3000" environment: - NODE_ENV=production - DB_HOST=db - DB_PASSWORD=secret depends_on: - db restart: always # 数据库 db: image: mysql:8.0 container_name: my-db environment: MYSQL_ROOT_PASSWORD: secret MYSQL_DATABASE: myapp volumes: - db_data:/var/lib/mysql restart: always # Redis 缓存 redis: image: redis:alpine container_name: my-redis restart: always volumes: db_data:

Compose 常用命令

# 启动所有服务（后台） docker compose up -d # 启动并重新构建 docker compose up -d --build # 查看运行状态 docker compose ps # 查看日志 docker compose logs # 查看特定服务日志 docker compose logs -f web # 停止服务 docker compose stop # 删除容器和网络 docker compose down # 删除容器、网络和卷（慎用） docker compose down -v # 重启服务 docker compose restart # 进入容器 docker compose exec db mysql -uroot -p

💡 实际应用场景

假设你要搭建一个 WordPress 博客系统，需要 Nginx + PHP + MySQL，用 Compose 只需要一个命令就能启动整个环境！

❓ 常见问题

🔧 安装问题

Q: Windows 提示 "WSL2 未安装"

解决：以管理员运行 PowerShell：

wsl --install dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart

Q: Linux 执行 docker 命令需要 sudo

解决：将用户加入 docker 组后重新登录

sudo usermod -aG docker $USER newgrp docker

🐛 运行问题

Q: 端口被占用 (port is already allocated)

解决：更换主机端口或停止占用端口的程序

# 查找占用端口的进程 sudo lsof -i :80 # 使用其他端口 docker run -d -p 8080:80 nginx

Q: 容器启动后立即退出

可能原因：前台进程结束，容器认为任务已完成

解决：确保容器有持续运行的前台进程

# 查看退出原因 docker logs 容器ID # 交互式调试 docker run -it --rm 镜像名 /bin/sh

Q: 无法连接容器内的服务

检查清单：

  • 是否正确映射了端口 -p 主机端口:容器端口
  • 容器内的服务是否监听 0.0.0.0（不是 127.0.0.1）
  • 防火墙是否放行了端口

💾 数据问题

Q: 容器删除后数据丢失

原因：容器内的数据是临时的

解决：使用卷（Volume）持久化数据

# 使用命名卷 docker run -d -v mydata:/data nginx # 使用主机目录 docker run -d -v /host/path:/container/path nginx # 查看所有卷 docker volume ls

🧹 清理问题

# 查看磁盘使用 docker system df # 清理停止的容器 docker container prune # 清理未使用的镜像 docker image prune -a # 清理未使用的卷（慎用） docker volume prune # 一键清理所有（极度慎用） docker system prune -a --volumes
