git config

配置 Git 环境变量，包括用户名、邮箱、编辑器偏好等。

# 设置全局用户名
git config --global user.name "Your Name"

# 设置全局邮箱
git config --global user.email "email@example.com"

# 设置默认编辑器
git config --global core.editor "vim"
配置 入门

git config --list

查看当前所有的 Git 配置信息。

# 列出所有配置
git config --list

# 查看特定配置
git config user.name

git init

在当前目录初始化一个新的 Git 仓库，创建 .git 子目录。

# 初始化仓库
git init

# 初始化裸仓库（用于服务器）
git init --bare project.git
初始化

git clone

从远程仓库克隆项目到本地，自动设置远程连接。

# 克隆仓库
git clone https://github.com/user/repo.git

# 克隆指定分支
git clone -b develop https://...

# 浅克隆（最近历史）
git clone --depth 1 https://...

git status

检查工作区文件状态，显示已修改、已暂存和未跟踪的文件。

# 简短格式
git status -s

# 分支信息
git status -b
常用 状态

git add

将文件更改添加到暂存区（Index），准备提交。

# 添加所有更改
git add .

# 添加特定文件
git add filename.txt

# 交互式添加
git add -p

git commit

将暂存区的更改保存到本地仓库，创建新的提交记录。

# 基本提交
git commit -m "feat: add login function"

# 修改最后一次提交
git commit --amend

# 跳过暂存直接提交已跟踪文件
git commit -a -m "message"

git log

查看提交历史记录，支持多种格式和筛选选项。

# 图形化日志
git log --oneline --graph --all

# 查看文件历史
git log -p filename

# 统计信息
git log --stat

git diff

显示工作区与暂存区、暂存区与最新提交之间的差异。

# 查看未暂存的更改
git diff

# 查看已暂存的更改
git diff --staged

# 查看特定文件的差异
git diff filename

git rm / mv

从 Git 中删除或移动文件，并记录这些更改。

# 删除文件并暂存
git rm filename

# 仅停止跟踪但保留文件
git rm --cached filename

# 移动/重命名
git mv old new

git branch

列出、创建或删除分支。分支是 Git 的核心特性之一。

# 列出本地分支
git branch

# 创建新分支
git branch feature-x

# 删除已合并分支
git branch -d feature-x

# 强制删除
git branch -D feature-x
分支

git checkout

切换分支或恢复工作区文件。Git 2.23+ 推荐使用 switch/restore。

# 切换分支
git checkout main

# 创建并切换
git checkout -b feature-x

# 切换到提交（分离 HEAD）
git checkout abc123

git switch / restore

Git 2.23+ 引入的实验性命令，更清晰分离切换分支和恢复文件的功能。

# 切换分支（替代 checkout）
git switch main

# 创建并切换
git switch -c feature-x

# 恢复文件（替代 checkout --）
git restore filename

git merge

将指定分支的更改合并到当前分支，可能产生合并提交。

# 合并分支
git merge feature-x

# 压缩合并（创建一个提交）
git merge --squash feature-x

# 快进合并
git merge --ff-only feature-x

git rebase

变基操作，将提交移动到新的基础提交上，用于保持线性历史。

# 变基到主分支
git rebase main

# 交互式变基（修改历史）
git rebase -i HEAD~3

# 继续变基（解决冲突后）
git rebase --continue

git remote

管理远程仓库连接，查看、添加、删除远程仓库地址。

# 查看远程仓库
git remote -v

# 添加远程仓库
git remote add origin https://...

# 修改远程 URL
git remote set-url origin https://...
远程

git push

将本地分支的提交上传到远程仓库。

# 推送到远程
git push origin main

# 推送所有分支
git push --all origin

# 强制推送（谨慎使用）
git push -f origin main

# 设置上游分支
git push -u origin main

git pull

从远程仓库获取最新更改并合并到当前分支（fetch + merge）。

# 拉取并合并
git pull origin main

# 拉取并变基（推荐）
git pull --rebase origin main

# 仅快进
git pull --ff-only

git fetch

从远程仓库下载最新更改，但不自动合并，允许你先审查更改。

# 获取所有远程分支
git fetch origin

# 获取特定分支
git fetch origin main

# 清理已删除的远程分支
git fetch --prune

git reset

重置当前 HEAD 到指定状态，用于撤销提交（会修改历史）。

# 软重置（保留更改到暂存区）
git reset --soft HEAD~1

# 混合重置（默认，保留更改到工作区）
git reset --mixed HEAD~1

# 硬重置（丢弃更改，危险！）
git reset --hard HEAD~1
危险 撤销

git revert

创建一个新的提交来撤销指定提交的更改，安全地撤销已推送的提交。

# 撤销最后一次提交
git revert HEAD

# 撤销特定提交（不自动提交）
git revert -n abc123

# 撤销合并提交
git revert -m 1 abc123

git clean

从工作区删除未跟踪的文件（未被 Git 管理的文件）。

# 查看将被删除的文件（干运行）
git clean -n

# 删除未跟踪的文件和目录
git clean -fd

# 包括被忽略的文件
git clean -fdx

git reflog

记录所有分支的更新记录，是后悔药，可找回丢失的提交。

# 查看引用日志
git reflog

# 恢复到特定状态
git reset --hard HEAD@{2}

# 查看特定分支的 reflog
git reflog main

git stash

临时保存工作区的更改，让你可以切换到其他分支工作。

# 暂存更改
git stash push -m "描述信息"

# 包含未跟踪文件
git stash -u

# 查看暂存列表
git stash list

# 恢复最近的暂存
git stash pop

# 应用到工作区但不删除
git stash apply
暂存

git tag

创建标签用于标记特定的提交，通常用于版本发布（v1.0, v2.0）。

# 创建附注标签
git tag -a v1.0 -m "版本 1.0"

# 创建轻量标签
git tag v1.0-lw

# 推送标签到远程
git push origin v1.0

# 推送所有标签
git push origin --tags
版本

git cherry-pick

选择性地将特定提交的更改应用到当前分支，而不合并整个分支。

# 挑选单个提交
git cherry-pick abc123

# 挑选多个提交
git cherry-pick abc123 def456

# 挑选范围
git cherry-pick abc123^..def456
高级

git bisect

二分查找工具，用于快速定位引入 Bug 的提交。

# 开始二分查找
git bisect start

# 标记当前为坏提交
git bisect bad

# 标记已知的好提交
git bisect good v1.0

# 结束查找
git bisect reset

git worktree

管理多个工作树，允许同时检出多个分支到不同目录。

# 为分支创建新工作目录
git worktree add ../feature-branch feature

# 列出工作树
git worktree list

# 删除工作树
git worktree remove ../feature-branch

git submodule

管理嵌套的 Git 仓库，允许在一个项目中包含其他项目。

# 添加子模块
git submodule add https://... path

# 初始化并更新
git submodule update --init --recursive

# 更新子模块
git submodule update --remote
子模块

没有找到匹配的命令

git commit -m "Stay curious, keep coding"
