网络架构核心概念

正向代理 VS 反向代理

深入理解两种代理模式的本质区别、工作原理及应用场景

Forward Proxy

正向代理

代理客户端，隐藏客户端身份，代表用户访问外部资源。

科学上网 / VPN
企业访问控制
内容缓存加速
形象比喻：房产中介
Reverse Proxy

反向代理

代理服务器，隐藏服务端身份，保护后端服务器集群。

负载均衡 (Nginx)
SSL 终端 / HTTPS
WAF 安全防护
形象比喻：酒店前台

核心维度对比

对比维度 正向代理 反向代理
代理对象 客户端 (Client) 服务器 (Server)
隐藏对象 隐藏客户端真实 IP 隐藏后端服务器信息
部署位置 客户端侧 / 客户端可访问 服务器侧 / 数据中心
使用者知情 需主动配置 对客户端透明
典型用途 翻墙、访问控制、缓存加速 负载均衡、安全防护、SSL终结
配置复杂度

工作流程可视化

正向代理流程

客户端
用户设备
知道代理存在
正向代理服务器
Forward Proxy
目标服务器
Google / 被屏蔽网站
只知道代理IP
关键特征：客户端主动配置代理地址，所有请求先发送到代理服务器，由代理转发到目标。

反向代理流程

客户端
浏览器 / App
以为是真实服务器
反向代理服务器
Nginx / CDN
Server 1
Server 2
Server 3
关键特征：客户端无感知，请求发送到代理，代理根据策略分发到后端集群，隐藏真实服务器。

正向代理适用场景

科学上网 / VPN

访问被地区限制或屏蔽的外部网站，如 Google、YouTube

企业访问控制

监控和限制员工访问特定网站，提高工作专注度

缓存加速

缓存常用资源，减少带宽消耗，加速重复访问

反向代理适用场景

负载均衡

将流量分发到多台服务器，避免单点故障，提升并发能力

SSL 终端 / HTTPS

统一处理 HTTPS 加密解密，减轻后端服务器计算压力

WAF 安全防护

防御 SQL 注入、XSS 攻击，隐藏真实服务器 IP

形象比喻理解

正向代理 = 房产中介

你想买房（访问网站），但找不到房源（被屏蔽），于是找中介（代理）。
中介知道你是谁，但卖家只知道中介，不知道你是真正的买家。

保护客户端隐私

反向代理 = 酒店前台

你打电话给酒店总机（访问网站），前台（代理）转接到具体房间。
你不知道房间号，也不知道酒店有多少间房（后端服务器），只知道总机号码。

保护服务端安全

技术实现示例

nginx.conf
# 正向代理配置
server
{
resolver 8.8.8.8;
listen 8080;
location /
{
proxy_pass $scheme://$host$request_uri;
}
}
# 反向代理配置
server
{
listen 80;
server_name example.com;
location /
{
proxy_pass http://backend_cluster;
proxy_set_header Host $host;
}
}
两者使用相同的 Nginx 软件，只是配置逻辑不同：proxy_pass 指向外部为正向，指向内部 upstream 为反向

一句话总结

正向代理 是客户端的"代言人"，隐藏用户身份访问外部世界；
反向代理 是服务器的"门面担当"，保护后端集群并优化访问体验。
